始终没有答案：**回报在哪里？** 到底节省了多少人时？你的钱有多少**蒸发**在幻觉、返工和等待里？哪个 AI 项目**今天值得扩张**——而
哪个正在**烧掉现金**，你却还在为「创新」鼓掌？

你没有「AI 成本」，你有一处**无声的漏洞**——而且蒙着眼。因为*「无法度量的东西，就无法管理，也无法改进」*——市场会替你度量，并把
账单递给你。

## 🌅 为何这将改变格局

**在 AI 时代，人分两种。** 第一种订阅一切、大手笔花钱，然后**祈祷**成功——并壮大了那条残酷的统计：项目死在试点。第二种做的，是华尔街
对任何严肃资产所做的：**度量、比较、排序、再配置**——把每一美元订阅变成**复利回报**。她们之间唯一的差别**不是天赋，也不是预算，
而是「仪表化」。**

生成式 AI 造出了一类新的经常性支出——**订阅与 token**——并随之带来了这十年最昂贵的浪费：**隐形的浪费。** 看不见，就修不了；不度量，
就扩不了；证明不了，董事会就不批。

**这个项目把你从第一部落带到第二部落。** 它把每一个 AI 项目当作**金融资产**来仪表化，并在**平衡计分卡**、**投资分析（NPV、IRR、
MIRR、EAA、PI、回收期）**和 **精益六西格玛** 下度量它——甚至用多准则模型（**AHP-TOPSIS 2n**）**选出你投资组合中最好的项目**。
不透明的月度账单变成一份**可审计的投资论证**：你用数字看清在哪里扩张、在哪里砍掉、订阅在哪几**周**内回本——又在哪里毫无遮盖地
失血。

我们是新疆域的**拓荒者**——**人工智能与高价值金融智慧之间的边界**。像绘制未标记之地的探险家，这个框架就是那枚**指南针**（🧭），把订阅的
迷雾化为**清晰的回报航线**：每个 token 是一英里，每个项目是一次奔向利润的远征。原本盲目的成本处，诞生**可度量的机会**；原本
一张死表格处，绽放**鲜活的投资论证**。

### 🚀 写给 Micro-SaaS、SaaS、初创公司与独立创业者

当你把 AI 嵌进产品的那一刻，没人告诉你一件事：**你刚刚把 AI 从市场预算搬进了你的 COGS（销货成本）。**
而随使用量增长的 COGS 不是费用 —— 它是**押在你毛利上的一笔按揭**。每多一个用户就多烧 token；每一次因幻觉导致的重试，
都把毛利烧掉两遍。而账单只在月底才现身，那时已无从挽回。

| 你是…… | 没人度量的那处痛 | 这个框架还给你的 |
|---|---|---|
| **独立创业者** | 你*就是*团队；你的一小时是世上最贵的资产 | **龙卷风图**指出真正牵动结果的变量 —— 也就是**下一个小时该投在哪里** |
| **Micro-SaaS** | token 成本随使用增长，悄无声息地吞掉毛利 | 用**你真实 token 拟合出的分布** + **CVaR**：糟糕的月份在到来*之前*就已标好价 |
| **规模化 SaaS** | 每个 AI 功能都是一个项目，抢的是同一份路线图 | **五种方法**选出该上哪一个 —— 而**稳健性**告诉你第一名能否扛住权重上 2 个百分点的误差 |
| **正在融资的初创** | 投资人不会为*「我们用了 AI」*买单 | 他买的是 **NPV、IRR、回收期与 `P(NPV<0)`** —— 而且 **pitch deck 直接生成**，LaTeX 排版 |

**真正改变的是什么。** 你的毛利不再是一个估计值，而成为一条**尾部已被定价的分布**。你的 runway 不再是简单除法，
而多了一个 **VaR**：*「在 20 个情形中的 19 个里，我的现金至少能撑 N 个月。」* 到了董事会材料或尽职调查的时刻，
你打开的不是一张没人复现得了的表格 —— 而是一个**固定随机种子**的数字，任何合伙人、投资人或审计师重跑一遍，
得到的**分毫不差**。

> **一次残酷的重新定位：** 独立创业者开始像 CFO 一样决策；而 CFO 开始以独立创业者的速度决策。

> **承诺：** 把*为 AI 付费的人*变成*用 AI 赚钱的人*——把*使用 AI 的人*变成**开创性地驾驭、度量并放大它的人**。用数字，而非用信念。

## 📈 证据（Gartner · IDC · PwC · McKinsey · MIT）

别只信我。**去信那些研究此事数十年的机构**——它们的结论一致：**AI 创造巨大价值，但只交付给那些度量并治理的人。** 「用 AI 却不
驾驭 AI」的人沦为放弃统计；而仪表化回报的人**赢得奖赏**。

- 🧭 **Gartner** — 预测**到 2025 年底，≥ 30% 的生成式 AI 项目会在概念验证后被放弃**，核心原因是**业务价值不清晰**（此外还有劣质
  数据、成本攀升与控制薄弱）。*→ 没有度量，项目就死在试点。*
- 🔬 **MIT**（报告 *「The GenAI Divide / State of AI in Business 2025」*，NANDA 计划）— 广泛报道称**绝大多数企业 GenAI 试点
  对损益没有可度量的影响**；少数交付价值者，是把 AI 与**流程和度量**结合。*→ 差别在于度量，而非采用。*
- 💵 **IDC**（研究 *「The Business Opportunity of AI」*，由微软赞助）— **度量并优化**的组织报告，每投入 **1 美元** AI 就有
  **数美元**量级的回报，且领先者与落后者之间差异巨大。*→ ROI 确实存在——并偏爱仪表化的人。*
- 🌍 **PwC**（*「Sizing the Prize」*）— 估计到 2030 年 AI 可为全球经济增加高达 **~15.7 万亿美元**；但奖赏归于**捕获**价值者，
  而非仅仅消费者。*→ 蛋糕巨大，而那块属于度量的人。*
- 🏆 **McKinsey**（*「The State of AI」*）与 **BCG × MIT Sloan** — 少数派 **「AI high performers」** 捕获了不成比例的回报；
  转折点在于把 AI 与已被证明有回报之处的**指标、治理与再投资**耦合起来。*→ 赢家度量、排序并再配置。*

> **这道鸿沟，正是本框架要跨越的：** 它把你从*在试点放弃*的一侧，带到拥有**实打实且经证明结果**的一侧——用 BSC、投资分析与多准则决策。

> ⚠️ **诚实声明（请阅读）：** 上述数字反映这些机构的真实报道，但**报告与百分比会更新**——在正式材料中引用前，请到**原始来源**
> （Gartner Newsroom；IDC/Microsoft *Business Opportunity of AI*；PwC *Sizing the Prize*；McKinsey *State of AI*；
> MIT *State of AI in Business*）核实确切数值与年份。此处它们用作**方向性依据**，而非数字保证。

## 💥 不作为的代价（算一算没人算的那笔账）

一份 **AI PRO** 订阅每月每席位在 **20 至 200 美元**之间。乘以团队人数，再乘以 12 个月。现在套用机构**已经证明**的事实：**Gartner**
预测**≥ 30% 被放弃**，**MIT** 显示**多数试点没有回报**。这笔总额中很大一块不是投资——是**纯粹的失血**。

> **直接举例（换成你自己的数字）：** 10 席位 × 30 美元/月 × 12 = **3,600 美元/年**。若 ~30% 变成隐形浪费，就是
> **~1,080 美元/年在蒸发**——仅一个小团队，仅一年。换成你的真实数字，惊吓更大。

而最扎心的部分是：**这项成本会复利，且不会等你。** 每一个不度量的月份，都是一个**回不来**的漏损月份——而那位已仪表化的竞争者，
**早已把资本再配置到能赚钱的地方**。先发优势要趁早建立：**谁先度量，谁先扩张。**

开始的最低成本时刻是昨天。第二好的是**现在**——而它只花 **0 美元**和 **5 分钟**。问题不是*「我付得起去度量吗？」*，而是
***「我还能付得起多久『不』度量？」***

## ✨ 功能

- **📊 KPI（BSC 四视角）+ API 经济学：** 成熟度、人力资本、财务与 token 效率——`IEET`、`IOLI`、`ITR`、`IITA`、`PEUC`、`ICCA`、
  `IDLS`、`IBMT`——外加 **EVM**（CPI/SPI/EAC）。
- **🪙 前沿概念：** **VRT/kTR**（可代币化的成本回收单位——*「Gitman 的每平方米」*）与 **PSR**（Project Score 0–5 ⭐），用于给每个
  项目的健康度排名。
- **🔬 运营诊断：** **VRT 五区块**、**HCI**（关键中断时刻）、**精益六西格玛浪费**（加权 token）与**按提示词分类学的幻觉 RCA**
  （每项目瓶颈 + 交集）。
- **💰 完整财务套件：** **NPV、IRR、MIRR、EAA（等额年金）、PI、回收期**（简单与折现）、**美元化**及与 **SELIC** 和**美国利率**的对比。
- **🏆 多准则决策：** **AHP-TOPSIS 2n**（双重归一化）以**稳健性检验**选出组合中的**最佳项目**——并生成**管理档案**
  （SWOT、PESTELC、5W4H、Pareto、GUT、Radar）。
- **🗺️ C 级视觉：** **交互式 5D 地图**、带深度的甜甜圈图、可持续性象限、趋势，以及合格项目的 LaTeX **pitch deck**。
- **⚙️ 真实管线：** **Langfuse → SQLite → Evidence**，具备**异步并发**同步与 **Rust（PyO3）**加速的分类。
- **💳 AI FinOps：** **订阅计划**目录（OpenAI、Anthropic、Google、Perplexity、xAI、Mistral、DeepSeek、Kimi、Qwen……），含
  **汇率 + IOF** 与分摊基准（burn rate）。
- **🌐 12 种语言**——仪表盘**以及图表图片内的文字**（含天城文、希伯来文与 CJK）。
<!-- eixo-execucao -->
- **📅 蒙特卡洛进度（PERT）+ 甘特图：** 同一套蒙特卡洛引擎指向**任务工期** —— 完工日期分布、**P80**（PMI 推荐的承诺日期）、`P(按时完成)`，以及带**关键度指数**的**关键路径**（在 10,000 次模拟中该任务有多少百分比落在关键路径上 —— 确定性 CPM 掩盖的正是这一点）。
- **📐 挣值管理（EVM）+ 挣得进度（ES）：** **SPI · CPI · EAC · ETC · VAC · TCPI** 与 **S 曲线**（PV/EV/AC）。把**成本 + 进度 + 范围**统一在一张图里。**SPI(t)** 修正了 SPI 的已知缺陷 —— 项目临近结束时即使延期它也会收敛到 1。
- **⚙️ AI 执行健康度（随时间）：** 对照 **SLO** 的 **p50/p95/p99** 延迟、**token 预算燃尽**、**质量回归**（类 SPC 规则）与基于 **柯尔莫哥洛夫-斯米尔诺夫** 的**模型漂移** —— 全部来自 Langfuse 的**真实**日志。
- **🚨 风险登记册 + 概率 × 影响矩阵：** 每个 PMO 都要的*定性*风险，含责任人、触发条件与缓解措施。概率**锚定于真实信号**（SLO 违规、CPI、漂移），不是拍脑袋。
- **🌊 流动度量（看板）：** **CFD**、**周期时间 P50/P85**（按分位数预测，而非靠猜）、**吞吐量**与 **WIP**。

## 🎲 蒙特卡洛 — 平均值掩盖的风险

**平均**为正的 NPV 保护不了任何人。平均值是金融里最舒适的谎言：它描述的场景也许永远不会发生。决定你命运的是**尾部**——那个糟糕的日子。

本框架为每个项目模拟 **10,000 个未来**：每一笔现金流都成为
**随机变量**，整个组合逐次迭代重算。最后你得到的不是一个数字，而是**你金钱的完整分布**：

- **`P(NPV < 0)`** — 真实的亏损概率。没人给你看的那个数字。
- **VaR 5%** — 最坏的合理情形：*「在 20 个未来中的 19 个里，我至少赚这么多。」*
- **CVaR 5%** — 灾难真的发生时，平均要付出多少代价。
- **敏感性龙卷风图** — 多元回归与皮尔逊相关：究竟哪个变量在真正牵动你的 NPV。
- **20 种输入分布**、经校验的**相关矩阵**（Iman-Conover，精确保留边缘分布）以及 **1% 至 99% 的百分位**，
  以及 100 类直方图。

固定随机种子：再跑一次得到**完全相同**的结果。可审计——而非「魔法」。

> **转折点：** 你不再挑 NPV 最高的项目，而是挑**在坏情形下仍能存活的项目**。这就是风险管理——投资者与赌徒的分界线。

## 🔬 信号在上游 —— 杠杆就藏在那里

这是我测量框架本身时发现的：NPV 的敏感性龙卷风图返回的竟然**恰好**是
`1.0 · 0.9091 · 0.8264 · 0.7513…` —— 折现因子 `1/(1+i)ᵗ`。因为 NPV 对现金流是**线性**的，只模拟现金流除了折现率之外
什么也告诉不了你。**真正的随机信号在上游：在 token 里。**

### 1️⃣ 别再拍脑袋定分布。把它拟合到你的数据上。

十一种候选分布以**最大似然**拟合你真实的 token 消耗序列（`logs_langfuse`）。**AIC 最小者**获胜 —— AIC 会惩罚每一个多余
参数、防止过拟合 —— 而 **Kolmogorov-Smirnov** 检验衡量拟合优度。这正是经典的*将分布拟合到数据*，
它揭示消耗的**厚尾**：有些提示词的成本是典型值的 10 倍，而正是这条尾巴撑爆预算 —— 对只看平均值的人完全不可见。

**而当拟合很差时，框架会大声喊出来。** 若 KS 的 p 值低于 0.05，界面会以红色标注 `拟合不佳`，而不是假装精确。
一个诚实的数字胜过一个漂亮的数字。

![将分布拟合到真实 token —— 11 个候选、按 AIC 选择、以 Kolmogorov-Smirnov 检验拟合优度](docs/screenshots/ajuste-distribuicoes.png)

### 2️⃣ 权重差 2 个百分点，你的排名还站得住吗？

任何多准则方法给出赢家时，都隐含着**100% 的置信度**。但准则权重是估计值，不是被揭示的真理。如果 IITA 权重上的两个百分点
就能让第一和第二互换，那么这个「赢家」不过是标定的产物。

于是我们用 **Dirichlet** 扰动 DEMATEL 的权重 —— `w' ~ Dir(κ·w)`，它恰好生活在单纯形上并保持 `E[w'] = w`，
即**无偏扰动** —— 然后重排 **2,000 次**。裁决的性质就变了：

> *「Project C 最好」* ⟶ **「Project C 在 99.9% 的合理偏好宇宙中胜出」**

这是一个**关于决策本身的置信区间**。它还揭穿了共识所掩盖的事：在下图中，**PROMETHEE II 仅在 25.4% 的宇宙里选出该领先者**。
四个学派同意，一个正面反对。这不是噪声 —— 这是警示：你的选择取决于你更偏爱*级别高于关系*还是*效用*。
任何单一排名都不会告诉你这一点。

## 🎓 基础：什么是蒙特卡洛，什么是多准则决策

### 🎲 蒙特卡洛数学模拟

#### 📖 概念

蒙特卡洛是一种**通过随机抽样**回答不确定系统问题的数值方法。它的思路颠倒了数学家的本能：与其推导**封闭解**——去解那个积分、
那个组合数、那个微分方程——不如构造一个**模型**，抽取成千上万次不确定变量的实现，然后**数一数发生了什么**。最后得到的不是一个
数字，而是**结果的概率分布**。

两条定律支撑着它。**大数定律**保证模拟的均值收敛于真值；**中心极限定理**告诉你收敛有多快：标准误以 `1/√N` 下降。
其推论既诚实又有几分残酷——**要把精度翻一番，就得把迭代次数翻两番**。蒙特卡洛并不快。它的美德在别处：误差**与问题的维度无关**。
确定性求积方法受制于*维度诅咒*，几十个变量便崩溃；蒙特卡洛不会。它恰恰赢在解析数学死去的地方。

#### 📖 何时何地诞生

**1946 年，新墨西哥州洛斯阿拉莫斯。** 波兰数学家**斯坦尼斯瓦夫·乌拉姆（Stanisław Ulam）**正从脑炎中康复，成天玩纸牌接龙。
他想知道一局获胜的概率是多少。他试了组合数学，随即放弃：根本算不动。接着他想到一个简单得近乎作弊的办法——**玩上一百局，
数出赢了几局，然后相除**。他立刻意识到，这不是牌桌上的小把戏：这是一种面向无人会解的问题的**通用积分方法**。

他把这个想法带给**约翰·冯·诺伊曼（John von Neumann）**，后者立刻看出它适用于曼哈顿计划中困扰他们的问题：可裂变材料中的
**中子扩散**。模拟成千上万个中子的随机行走——散射、吸收、裂变——是可行的；解输运方程则不然。**尼古拉斯·梅特罗波利斯
（Nicholas Metropolis）**提议将其命名为**「蒙特卡洛」**，取自乌拉姆的舅舅常借钱去豪赌的摩纳哥赌场。**ENIAC** 让最初的计算成为可能；
**1949 年**，梅特罗波利斯与乌拉姆在《美国统计学会杂志》上发表了*《The Monte Carlo Method》*。

这个方法，字面意义上诞生于**纸牌游戏**与**原子弹**的相遇。很少有科学思想拥有如此令人错愕的出生证明。

#### 📖 方法论的五个步骤

1. **建模。** 把输出写成输入的函数：`y = f(x₁, …, xₙ)`。
2. **赋予分布。** 每个不确定的输入都要有一个分布。若有**历史数据**，就把分布*拟合*到数据上；若没有，则由人指定——而这必须**明确声明**。
3. **抽样。** 抽取 `N` 个情景。若变量之间**相关**，抽样必须尊重这一结构：在真实存在依赖之处独立抽样，是本方法最常见的错误。
4. **传播。** 在每个情景中计算 `f`。正是在这里，输入的不确定性**转化**为输出的不确定性，且不作任何线性近似。
5. **分析。** 研究所得分布：均值与标准差、**百分位**、事件概率（`P(y < 0)`）、**尾部**（VaR、CVaR）以及**敏感性**（哪个 `xᵢ` 牵动 `y`）。

#### 📖 世界上的用途与应用

- **金融：** 奇异期权定价（无封闭解之处）、组合的 **VaR** 与 **CVaR**、监管压力测试（巴塞尔协议）、信用风险。
- **工程：** 结构可靠性、制造公差、复杂系统的失效分析。
- **项目管理：** 工期与成本风险（PERT 的概率化演进）、完工 S 曲线。
- **物理与化学：** 粒子输运、辐射屏蔽、统计力学。
- **运营与供应链：** 排队、库存、需求不确定下的产能。
- **流行病学：** 疾病传播与不确定条件下的政策评估。
- **AI 自身之内：** **MCMC**（贝叶斯推断）、**MCTS**——让 AlphaGo 战胜李世石的那套树搜索——以及用于估计神经网络不确定性的
  *Monte Carlo dropout*。

#### 🔒 本项目中独有的方法论、用途与应用

在这里，蒙特卡洛不是学术装饰：它是投资组合的**风险引擎**。

- **输入。** 每一笔周期性现金流都成为**三角分布**变量（`min`、`众数`、`max`），众数取确定性值，两尾各 ±30%。而 **token 消耗**——
  唯一真正厚尾的变量——**并非人为指定**：十一个候选分布以**最大似然**拟合到你真实的 `logs_langfuse` 序列，**AIC 最小者**胜出，
  拟合优度以 **Kolmogorov-Smirnov** 检验衡量。若 p 值跌破 0.05，界面会以红色打出 `拟合不佳`，而不是假装精确。
- **相关性。** 当现金流彼此依赖时，抽样采用 **Iman-Conover**：它施加秩相关，同时**精确保留边缘分布**。矩阵事先经过校验：对称、
  对角为 1、正定。
- **传播。** 每个项目 **10,000 次迭代**，**固定随机种子（42）**：再跑一次，得到的数字分毫不差。这不是细节——正是它让结果可以被
  合伙人、投资人或审计师**审计**。
- **输出。** 不只是 NPV：我们模拟 **NPV、IRR、MIRR、EAA、PI** 以及 **token 成本**，每一项都附有十个经典描述统计量
  （偏度与峰度采用 Excel 的定义）、**1% 至 99% 的百分位**，以及 **100 类的直方图**。
- **真正重要的风险。** `P(NPV < 0)` 是亏损的真实概率。**VaR 5%** 是最坏的合理情形——*「在 20 个未来中的 19 个里，我至少赚这么多」*。
  **CVaR 5%** 回答了没人问的那个问题：灾难真的降临时，**平均要付出多少代价**。
- **敏感性。** **龙卷风图**以两种经典形式计算：**多元回归的 beta**（某个输入每增加 1 个单位对 NPV 的影响）与
  **皮尔逊相关系数**（该输入的不确定性在多大程度上支配了 NPV 的不确定性）。二者互补；仪表盘同时呈现。
- **框架对自身的一个发现。** 在度量自己时，龙卷风图返回的 beta **恰好等于折现因子** `1/(1+i)ᵗ`——因为 NPV 对现金流是*线性*的。
  因此，只模拟现金流除了折现率之外什么也说明不了。**真正的随机信号在上游，在 token 里。** 正是这个发现，促成了将分布拟合到真实数据。
- **风险反哺决策。** 蒙特卡洛的两个输出作为**准则**进入多准则模型：`P(NPV<0)` 作为**成本型**准则，**VaR 5%** 作为**效益型**准则。
  于是最终的选择从诞生之初就已**经过风险调整**，而不只是对期望值的调整。


**它是什么。** 一种用**抽签**回答难题的方法。与其去求解一个不确定系统的封闭数学解 —— 那往往不可能 —— 不如给输入变量赋予
**概率分布**，抽取成千上万个情景，逐一算出结果，然后观察输出的**整条分布**。大数定律保证收敛；误差以 `1/√N` 下降，
也就是说**把迭代次数翻两番，误差减半**。

**它如何诞生。** 洛斯阿拉莫斯，1946 年。**斯坦尼斯瓦夫·乌拉姆（Stanisław Ulam）**在病中康复期间玩纸牌接龙，
忽然想知道获胜的概率是多少。他意识到：解组合数学极其残酷 —— 但**模拟**几百局并数一数，却微不足道。
他把这个想法带给**冯·诺伊曼（John von Neumann）**，两人随即将其用在曼哈顿计划中困扰他们的问题上：
可裂变材料中的**中子扩散**。**尼古拉斯·梅特罗波利斯（Nicholas Metropolis）**为该方法取名「蒙特卡洛」，
取自乌拉姆的舅舅常去豪赌的摩纳哥赌场。**ENIAC** 让最初的计算成为可能。这个方法，字面意义上诞生于纸牌游戏与原子弹的相遇。

**今天用在哪里。** 金融中的期权定价与 **VaR**；工程中的结构可靠性；项目管理中的工期与成本风险；粒子物理；供应链；
流行病学。而在 AI 内部：**MCMC**（贝叶斯推断）与 **MCTS** —— 正是那套让 AlphaGo 战胜李世石的树搜索。

**它在这里为我们做什么。** 你项目的每一笔现金流都变成随机变量，而 token 消耗则采用**拟合自你真实数据**的分布。
我们跑 10,000 个情景；最后你拿到的不是一个 NPV —— 而是**你金钱的整条分布**：`P(NPV < 0)`（真实的亏损概率）、
**VaR 5%**（最坏的合理情形）、**CVaR 5%**（灾难真的发生时要付多少代价），以及**龙卷风图**（哪个变量真正牵动结果）。
平均值会撒谎；尾巴才做决定。

### 🧮 多准则决策分析（MCDA）

#### 📖 概念及其用途

在项目之间取舍之所以困难，有两个电子表格解决不了的原因。其一，准则彼此**冲突**：NPV 最高的项目往往风险也最高。其二，它们**不可通约**：
不存在任何诚实的算术，能把人民币、幻觉百分比与返工小时数加在一起。

MCDA（*Multi-Criteria Decision Analysis*，多准则决策分析）正是应对此事的学科——诞生于 1960—70 年代，处在运筹学与决策理论的交界。
它**不承诺给出正确答案**，它承诺一件更有用的事：让这个选择变得**明确、可审计、可辩护**。

它的奠基命题既令人不适又令人解脱：**脱离语境的「最好」并不存在。** 存在的是*在某人已明确写出的偏好体系之下*的最好。
每一个决策者其实都已在使用某套偏好体系——区别在于，没有 MCDA 时，它是**隐含的、不一致的、无法审计的**。
用一个显式模型换掉那份默会的意见——这就是全部的收获。

#### 📖 世界上的用途与应用

供应商遴选与项目组合排序；能源技术选择（光伏 × 风电 × 生物质）；工厂、医院与填埋场的选址；环境影响评价；公共政策与预算分配；
人员选拔；维修优先级；以及日益增多的新兴技术**技术经济评价**——而 AI 项目组合正属此列。

#### 📖 三大决策学派

- **美国学派（价值与效用）。** 把一切汇总成**单一数字**。它假定某一准则上的差评可以被别处的好评**补偿**。简洁、有力——有时也危险。
  `AHP`、`MAUT`。
- **欧洲学派（级别高于关系）。** 由 **Bernard Roy** 创立。它承认两个备选可能**不可比**，并允许**否决**：某一准则上的灾难性表现
  **不能**用别处的卓越来赎买。它以**阈值**刻画决策者真实的犹豫。`ELECTRE`、`PROMETHEE`。
- **建构主义学派。** 模型不是被*发现*的，而是**与决策者一同建造**的：通过问题结构化，以及锚定在参照水平上的标度。`MCDA-C`。

#### 📖 1. DEMATEL — *Decision-Making Trial and Evaluation Laboratory*

**它是什么。** 由 **Gabus & Fontela** 于 **Battelle 纪念研究所**（日内瓦，1972-73）创立，用以研究复杂而纠缠的世界性难题。
它**不给备选排名**：它绘制**准则之间的因果结构**。

**如何运作。** 专家填写**直接关系矩阵** `Z`（准则 *i* 对 *j* 的影响强度，0 到 4）。以 `s = max(最大行和, 最大列和)` 归一化，
得到**总关系矩阵** `T = X(I − X)⁻¹`——它把直接影响**与沿任意路径的所有间接影响**都加了进去。由此得出 `R`（行和）与 `C`（列和）：
**`R+C` 是显著度**（在系统中的重要性），**`R−C` 是关系度**（为正即**因**，为负即**果**）。

**一般用途。** 可持续供应链、技术采纳的障碍、系统性风险分析。

**🔒 在本项目中。** DEMATEL 回答的是**先于**排名的那个问题：*「我该在哪里发力？」*。它揭示 **IITA（幻觉）、PSR（健康度）与
IDLS（精益浪费）是因**，而 **NPV、IRR、PI 以及风险指标是果**。这既反直觉又令人解脱：追着 NPV 跑毫无意义——它只是一支**温度计**。
在幻觉上发力，NPV、IRR 与风险会*一起*变好。此外，准则的**权重**并非人为指定：它们**由影响结构导出**，
`wᵢ ∝ √((R+C)ᵢ² + (R−C)ᵢ²)`。这些权重供给**其余五种方法**——这正是 John (2025) 所描述的集成范式。

#### 📖 2. AHP-TOPSIS 2n — *层次分析法* + *逼近理想解排序法*

**它是什么。** **Saaty (1977)** 提出 AHP：以准则间的**两两比较**导出权重，并用**一致性检验**揭穿自相矛盾的判断（`CR ≤ 0.10`）。
**Hwang & Yoon (1981)** 提出 TOPSIS：最优备选应当**离理想解最近**、**离负理想解最远**。

**如何运作。** 归一化决策矩阵，将各列乘以权重，计算到理想解与负理想解的欧氏距离，再以**贴近度系数** `Ci = d⁻/(d⁺+d⁻)` 完成排序。

**一般用途。** 这是全世界 MCDM 中最常用的组合——从供应商遴选到绩效评价。

**🔒 在本项目中。** 我们在**两种归一化**下运行 TOPSIS——向量式（欧氏）与 min-max（线性）——这便是 **「2n」** 的由来。
每个项目得到两个系数，最终排名取其平均。由此我们获得一个几乎无人报告的量：**两种归一化之间的一致度**。
当二者对某个项目的名次意见相左，说明该项目的结果**对一个任意的技术选择很脆弱**——仪表盘会把这一点显示出来。
本项目的 Saaty 矩阵 `CR = 0.0119`，远低于 0.10 的门槛。

#### 📖 3. ELECTRE I — *ÉLimination Et Choix Traduisant la REalité*

**它是什么。** **Bernard Roy (1968)** 于巴黎 SEMA 咨询公司提出。它是欧洲级别高于关系学派的原点。它问的不是*「每个人几分？」*，
而是*「**a** 是否至少和 **b** 一样好？」*。

**如何运作。** 对每一对 `(a, b)` 计算两个指标。**一致性指数** `C(a,b)` 把 `a` 不劣于 `b` 的那些准则的权重相加；
**不一致性指数** `D(a,b)` 度量 `a` 相对 `b` 的**最大劣势**。当一致性高**且**不一致性低时，称 `a` **胜过** `b`。
**没有任何备选能胜过**的那一集合，就是**核**（*kernel*）——可辩护选项的清单。

**一般用途。** 公共与环境决策，在那里用别的准则去补偿一个灾难性准则是不可接受的。

**🔒 在本项目中。** ELECTRE 是那个**拒绝为了方便而说谎**的方法。一个 NPV 高到离谱、幻觉却触目惊心的项目，**买不到**它的位置：
**不一致性**会把它拦下。框架报告**核**——没有任何其他项目能胜过的那些——并以**净胜过程度**（它胜过多少个，减去多少个胜过它）为得分。
它也是六者中唯一被允许说出这句话的：*「这两个项目根本**无法比较**」*。

#### 📖 4. PROMETHEE II — *偏好排序组织法*

**它是什么。** **Jean-Pierre Brans (1982)**，后与 **Bernard Mareschal 和 Philippe Vincke (1985)** 一同精化。同属级别高于关系，
但有一个优雅的转折：不再是二元阈值，而是度量 `a` 相对 `b` 被偏好**多少**。

**如何运作。** 对每个准则，差值 `d = g(a) − g(b)` 经由一个**偏好函数**映射为 0 到 1 之间的程度。Brans 提出了**六种广义函数**
（通常型、准则型、偏好阈值型、水平型、带无差异的线性型、高斯型），由**无差异阈值 `q`**（低于它，差异无关紧要）与
**偏好阈值 `p`**（高于它，偏好为完全）来参数化。加权求和后：`φ⁺` 表示 `a` 胜过他者的程度，`φ⁻` 表示被胜过的程度，
而**净流** `φ = φ⁺ − φ⁻` 给出**完全预序**（PROMETHEE II）。

**一般用途。** 能源、物流、医疗——凡是需要刻画偏好**强度**之处。

**🔒 在本项目中。** 我们采用**带无差异的线性**函数，`q` 与 `p` 由每个准则上观测偏差的 10% 与 90% 分位数估得。
PROMETHEE 回答的是*「赢家好多少？」*，而不只是*「它更好吗？」*。也正是它带来了本组合中最有意思的发现：在稳健性分析里，
**PROMETHEE II 仅在 25.4% 的偏好宇宙中选出共识领先者**——而另外四个学派意见一致。共识**掩盖了一场学派之间的分歧**。

#### 📖 5. MAUT — *多属性效用理论*

**它是什么。** **Ralph Keeney 与 Howard Raiffa (1976)**，冯·诺伊曼与摩根斯坦的直接继承者。这是公理化形态的美国学派：
若你的偏好满足若干理性公理，则存在一个表示它们的**效用函数**，而决策就是**最大化期望效用**。

**如何运作。** 每个准则获得一个**效用函数** `uⱼ`，把绩效映射为满意度。总效用是可加的：`U(a) = Σ wⱼ · uⱼ(a)`——
其成立以偏好上的**可加独立性**为前提。关键在于函数的**形状**：**凹**的 `u` 代表**风险厌恶**（第二个一百万不如第一个值钱）；
线性即中性；凸性则是风险偏好。

**一般用途。** 医疗决策、能源政策、谈判——任何需要把对待风险的态度**明确写出并加以辩护**的场合。

**🔒 在本项目中。** 我们使用**指数**效用 `u(z) = (1 − e^(−r·z)) / (1 − e^(−r))`，厌恶系数 `r = 2`。
这是一次**公开声明的伦理抉择**：本框架是**保守的**。同样期望值之下，不确定的收益不如确定的收益值钱——
一位审慎的 CFO 正是这样评估的。TOPSIS 把一切收益视作可互换，而 MAUT **惩罚那种高企而不确定的承诺**。

#### 📖 6. MCDA-C — *多准则决策辅助 — 建构主义*

**它是什么。** 由 **Leonardo Ensslin、Gilberto Montibeller 与 Sandra Noronha (2001)** 形式化，根植于 Roy 与 Bana e Costa。
其前提是哲学性的：模型**并不先于**决策者而存在，它是**与他一同建造**的，分三个阶段——**结构化**（认知图、描述子）、
**评价**（价值函数、替代率）与**建议**。

**如何运作。** 每个准则获得一个带若干水平的**描述子**，其中两个水平充当锚点：**中性**水平（低于它，绩效便构成危害）
与**好**的水平（高于它，即为卓越）。价值函数被锚定：中性处 `V = 0`，好处 `V = 100`，并在区间之外自由**外推**。

**一般用途。** 组织绩效评价、公共管理，以及决策者需要**学习**自身问题的种种情境。

**🔒 在本项目中。** 由于没有与决策者进行结构化会谈，我们把两个水平锚定在组合的**观测四分位数**上：`中性 = Q1`，`好 = Q3`。
这保住了 MCDA-C 独有之处——它不只是**排序**，它还**分类**：`V < 0` 为**危及**，`0 ≤ V ≤ 100` 为**有竞争力**，
`V > 100` 为**卓越**。一个项目可以高居榜首，却仍落在危及区间。这一组方法里，没有第二个会这样告诉你。

#### 📖 为什么是五个方法，而不是一个

因为**每个学派犯错的方式都不同**，而单一方法交出的赢家自带**隐含的 100% 置信度**——那永远是谎言。
AHP-TOPSIS 补偿过度；ELECTRE 有时拒绝下判断；MAUT 取决于效用的形状；MCDA-C 取决于锚点。

我们用**同一套权重**（来自 DEMATEL）跑完五个方法，再以 **Borda 共识**收口。此时它们之间的分歧不再是麻烦，而成为**信息**：
当四个同意、一个正面反对时，那不是噪声——那是警示：你的选择**取决于你隐含采纳的那个决策学派**。

#### 📖 最后一问：这个裁决扛得住吗？

上面整座大厦都建在**权重**之上，而权重是**估计值**。如果 IITA 权重上的两个百分点就能让第一与第二互换，
那么这个「赢家」不过是**标定的产物**，而非组合的事实。

因此我们用 **Dirichlet** 扰动 DEMATEL 的权重——`w' ~ Dir(κ·w)`，它恰好生活在单纯形上并保持 `E[w'] = w`，
即**无偏扰动**——然后重排 **2,000 次**。裁决的性质随之改变：

> *「Project C 最好」* ⟶ **「Project C 在 99.9% 的合理偏好宇宙中胜出」**

这是一个**关于决策本身的置信区间**。有了它，框架便不再只度量**金钱**的风险，而开始度量**决策**的风险。


**它是什么、有什么用。** 当你在项目之间取舍时，准则彼此**冲突**（高 NPV 往往伴随高风险），且**不可通约**
（你怎么把人民币和幻觉百分比相加？）。MCDA 正是让这种取舍变得明确、可审计、可辩护的学科。它的奠基命题既令人不适又令人解脱：
**脱离语境的「最好」并不存在。** 存在的是*在你明确写出的偏好体系之下*的最好。用一个显式模型换掉隐式的意见 —— 这就是全部的收获。

**三大学派。** **美国学派**，价值与效用（AHP、MAUT）：把一切汇总成一个数字。**欧洲学派**，级别高于关系
（ELECTRE、PROMETHEE），源自 Bernard Roy：承认两个备选可能**不可比**，并允许**否决** —— 某一准则上的糟糕表现，
不能用别处的优异表现赎买。**建构主义学派**（MCDA-C）：模型不是被发现的，而是**与决策者一同建造出来的**。

| **DEMATEL** | Gabus & Fontela，Battelle（1972-73） | *「谁影响谁？」* | 区分**因**与**果**，并从影响结构本身导出**权重** | 揭示降低幻觉（IITA）是**因** —— 在那里发力，NPV、IRR 与风险会一起变好 |
| **AHP-TOPSIS 2n** | Saaty（1977）· Hwang & Yoon（1981） | *「谁离理想解最近？」* | 以两两比较定权，并有**一致性检验**（CR ≤ 0.10） | 在**两种归一化**下排名，并报告二者的一致度 |
| **ELECTRE I** | Bernard Roy（1968） | *「谁胜过谁 —— 谁能不被支配地存活？」* | **不可比性**与**否决**：一项糟糕准则无法被赎买 | 分离出无人能支配的项目**核**（kernel） |
| **PROMETHEE II** | Brans & Vincke（1985） | *「净偏好流是多少？」* | **六种偏好函数**，带无差异与偏好阈值 | 刻画一个项目好*多少*，而不只是好*不好* |
| **MAUT** | Keeney & Raiffa（1976） | *「什么能最大化决策者的效用？」* | 以凹效用函数建模**风险厌恶** | 惩罚不确定的收益 —— 审慎的决策者不会为它们付同样的价 |
| **MCDA-C** | Ensslin, Montibeller & Noronha（2001） | *「「好」的水平在哪，「中性」又在哪？」* | **锚定的价值函数**：中性处 `V=0`，好处 `V=100`，并允许外推 | 分类为**危及 / 有竞争力 / 卓越**，而不只是排序 |

**为什么是五个，而不是一个。** 每个学派犯错的方式都不同。单一方法给出的赢家自带**隐含的 100% 置信度** —— 那永远是谎言。
把五个都跑一遍，再以 **Borda 共识**收口，它们之间的分歧就变成了**信息**：当四个同意、一个正面反对时，那不是噪声 ——
那是警示：你的选择取决于你更偏爱*级别高于关系*还是*效用*。而对权重施加的 **Dirichlet 扰动**回答了最后一问：
*「第一名扛得住标定上 2 个百分点的误差吗？」*


### 🧪 四个齿轮：Iman-Conover、Kolmogorov-Smirnov、Dirichlet 与龙卷风图

上面两大方法立足于四个更小的零件——而诚实的模拟与漂亮的数字之间的分野，恰恰藏在它们之中。值得一识。

#### 🔗 Iman-Conover —— 施加相关性，**却不摧毁分布**

**它是什么。** 由 **Ronald Iman 与 William Conover (1982)** 提出。它解决了一个看似平凡实则不然的问题：
*当边缘分布并非正态时，如何抽取相关的变量？* 天真的做法——用 Cholesky 生成相关正态再作变换——会**扭曲边缘分布**。
而如果你刚刚把一个 LogNormal 拟合到自己的数据上，扭曲它，等于把刚做完的工作扔掉。

**如何运作。** 它是一次**按秩重排**，而非数值变换。先用**范德瓦尔登评分** `Φ⁻¹(i/(n+1))` 构造参照，并按列打乱；
计算 `P = chol(R)`（目标）与 `Q = chol(corr(M))`（参照的偶然相关）；构造 `S = M·(Q⁻¹P)ᵀ`。
然后把原样本的每一列**按 `S` 的秩重新排序**。由于只改变了已抽出数值的*次序*，**边缘分布保持精确**——分毫不差。

**一个精细而诚实的细节。** `R` 是那个*正态参照*的相关系数，而非结果的皮尔逊相关。所诱导的秩相关遵循正态 copula：
`ρ_S = (6/π)·arcsin(R/2)`。当 `R = 0.80` 时其值为 **0.7859**——这正是我们测试时测到的（0.786）。
这不是方法的错误；这就是它的数学。

**一般用途。** 金融风险（相关资产）、结构可靠性、拉丁超立方抽样。

**🔒 在本项目中。** 正是它让我们在相关化现金流的同时，**不牺牲**已拟合到你 token 上的那个分布。使用之前，矩阵先经校验：
对称、对角为 1、且**正定**（经由 Cholesky）。不一致的相关矩阵会被拒绝，并报出最小特征值——而不是悄悄产出毫无意义的数字。

#### 📏 Kolmogorov-Smirnov —— 你所**假设**的与数据所**诉说**的之间的距离

**它是什么。** 一种**非参数**的拟合优度检验。统计量简洁而优美：`D = sup |Fₙ(x) − F(x)|`，
即你数据的**经验分布函数**与你所提出的**理论分布**之间最大的垂直落差。在原假设下，`D` 的分布**与 `F` 究竟是什么无关**——
故称 *distribution-free*。

**一条方法论上的诚实保留。** 经典 KS 的 p 值假定 `F` 的参数是在**看到数据之前**就已固定的。当它们是**由同一批数据估计**而来
（正如这里，用最大似然），检验就变得**过于乐观**：它太容易接受。严格起见应当采用 **Lilliefors** 校正或**参数化自助法**。
因此我们把 KS 视作**诊断**，而非证明——只用它来**拒绝**糟糕的拟合，绝不用它来宣称某个拟合是「正确的」。

**一般用途。** 拟合优度；两样本比较（双样本 KS）；生产环境机器学习系统中的数据*漂移*检测。

**🔒 在本项目中。** 它衡量按 AIC 胜出的那个分布究竟在多大程度上描述了你的 token 序列。当 p 值跌破 0.05，
界面便以红色打出 **`拟合不佳`**——在演示组合里，这确实发生在其中一个项目上，而框架把它**显示出来**，而不是藏起来。
一个诚实的数字胜过一个漂亮的数字。

#### 🎲 Dirichlet 扰动 —— **决策的置信区间**

**它是什么。** **Dirichlet** 分布是**单纯形**上的天然分布：一组求和为 1 的正数向量——这恰恰就是权重向量。
它是多项分布的共轭先验，也是 Beta 分布的推广。

**为什么用它，而不是高斯噪声。** 给权重加上正态噪声会产生负值，并破坏「和为一」。Dirichlet 就生活在合法空间*之内*。
而且，参数化为 `w' ~ Dir(κ·w)` 后，它具有两条让它完美胜任的性质：`E[w'] = w`（**无偏**扰动）与
`Var(w'ᵢ) = wᵢ(1−wᵢ)/(κ+1)`（离散程度只由一个旋钮控制）。当 `κ → ∞`，它便坍缩回原始权重。

**一般用途。** 比例的贝叶斯*先验*；主题模型中的隐含狄利克雷分配（**LDA**）；Rubin (1981) 的**贝叶斯自助法**；
以及多准则决策中的权重敏感性分析。

**🔒 在本项目中。** 取 `κ = 200`，一个 13% 的权重大约摆动 **±2.37 个百分点**——这正是专家判断的合理误差幅度。
我们重排 **2,000 次**，为每个项目得到 `P(胜出)`。正是这个齿轮揭示了本组合中最令人不安的发现：共识是稳健的（99.9%），
但 **PROMETHEE II 仅在 25.4% 的宇宙里选出该领先者**。没有 Dirichlet，这场分歧将始终不可见。

#### 🌪️ 敏感性龙卷风图 —— 究竟哪个变量**真正**牵动结果

**它是什么。** 一张按绝对效应排序的水平条形图，回答：*在所有不确定的输入之中，哪些在牵动输出？*
名字来自它的形状——上宽下窄。

**两个看似相同、实则不同的度量。**
- 多元回归的 **beta** 回答：*「若这个输入增加 1 个单位，输出会增加多少？」* 它是一个**单位**效应，
  对该输入实际波动多大毫不在意。
- **皮尔逊相关**回答：*「输出的不确定性中，有多少是由这个输入支配的？」* 它已经把**不确定性的尺度**纳入其中
  （近似为 `β·σᵢ/σ_y`）。

一个变量可以有巨大的 beta，而相关却为零：它*本会*大幅牵动结果，但实际上**几乎不波动**。只报告其中之一，只是半个真相。

**一般用途。** 项目风险、金融模型、可靠性工程、模拟器标定。

**🔒 在本项目中。** 这里的龙卷风图做了一件稀罕事：它**揭发了模型自身的局限**。对 NPV 运行时，那些 beta
**恰好等于 `1/(1+i)ᵗ`**——即折现因子——因为 NPV 对现金流是*线性*的。此时回归型龙卷风图是**退化的**：
除了折现率之外它什么也说明不了。承载信号的是**相关系数**。而当 token 成本作为变量进入之后，它的 beta 给出 `−1/(1+i)ᵗ`
（成本以负号进入），相关却接近于零。两者合读，结论既精确又诚实：*「token 每多花 1 个单位，NPV 就少 0.91——
但在这个项目里，NPV 的不确定性并非来自 token。」* 单看其中任何一个度量，都说不出这句话。

## 💰 全局 Token 预算 —— 每个项目都是一个成本中心

**只存在一份预算：你所订阅的那个方案的预算。** 其余一切都从它**往下分**。每个项目都是一个**成本中心** —— 它**没有自己的钱**。它的额度是**全局预算的一片切片**，而这片切片会在**每次有项目加入或退出组合时自动重算**。**没有任何东西被创造；一切都是被瓜分。**

> **这修复的结构性 Bug。** 每个项目的 token 预算原本是 `消耗 × 1.10` —— **十个项目**全都精确等于 1.100。循环。自我正当化。**按照构造，没有任何项目可能超支。** *从自己的花销推导出来的预算不是预算：那是收据。* 如今额度来自真实的池子，**10 个项目里有 6 个超支**。

```text
   ASSINATURA DO PLANO / PLAN SUBSCRIPTION
              │
              ▼
   💰 BUDGET GLOBAL  ─────────  a quota mensal contratada. É FINITA.
              │
              ├── piso igualitário (50%)
              └── por VALOR entregue (50%)
              │

   🏷️ CENTRO DE CUSTO 1 … N  ──  a cota de CADA projeto
```

### 🍩 概念 —— 池子是共享且有限的

**概念。** Langfuse、CloudZero、Vantage 之流给你的是**按项目计费**，仿佛每个项目都有自己的水龙头。**并没有。** 只存在**一份签约方案**，月度额度是有限的，而**一个项目烧掉的每一个 token，都是另一个项目拿不到的 token**。这就是把**公地悲剧**搬到了 AI 预算上。

**方法论。** 全局预算来自合同：`席位 × 美元 × 汇率 × (1+IOF)` 加上固定基础设施，得出**月度 TCO** 与**每百万 token 的成本**。真实消耗来自日志，折算成**月度 run-rate**。由此得出**额度利用率**、**余量**与**池子耗尽日**。

**应用 —— 以及那个让人痛的数字。** **31% 的消耗是浪费**：每月 2900 万 token 烧在**失败且什么都没返回**的调用上（幻觉、限流）。这是**你全部合同余量的 4.7 倍**。说白了：**你会因为那些从未给出答案的调用，被逼着去买更大的方案。** 砍掉一半浪费，释放出来的容量比整个余量还多 —— **而且不必多花一分钱**。

### ⚖️ 自适应分摊与交叉补贴 —— 谁在替谁买单

**概念。** **按消耗**分摊是市场惯例，而它是**自我正当化**的：烧得最多的拿到最大额度，这等于**为浪费背书**。诚实的分摊应当按**交付价值（EV）**。

**方法论。** 每个成本中心的额度 = `均等下限（50%）+ 交付价值（50%）`，**只要 N 变化就重新调整** —— 新项目的 EV = 0，若没有下限它将拿到**零 token**，从而永远无法产出价值。**交叉补贴**就是「按消耗应得额度」与「按交付应得额度」之差。补贴之和**恰好为零**：这是转移，不是价值创造。

**应用。** 效率差距达 **68 倍**：Project F 每百万 token 交付 **642** 的价值；Project J 只有 **10**。分摊把账单摊开：**每月 R$ 3,431 —— 占 TCO 的 40% —— 正从高效者流向低效者，月月如此，且无人察觉。** 组合中最便宜的 Project F，**正在替 Project J 买单**。

![交叉补贴 —— 消耗多于交付者被补贴；交付多于消耗者替别人买单](docs/screenshots/budget-subsidio-cruzado.png)

### 🔒 定价化的资源争用 —— 把因果链应用到整个组合

**概念。** 因果链在**单个项目内部**连接：`漂移的 token → 风险 → 工期（P80）→ 金钱`。而这里连接的是**项目之间**：`一个项目的超额 → 池子耗尽 → 其他项目被扼流 → 他们的 P80 滑移 → 他们的延迟成本来结账`。

**方法论。** 它**同时**需要 FinOps（额度）、EVM（交付价值）、风险（暴露度）与模拟进度（P80）。正因如此**市场上没有任何工具做得到** —— 没有一个同时拥有这四台引擎。Langfuse 看见 token。Jira 看见任务。CloudZero 看见账单。**它们谁也说不出「项目 J 正让项目 F 付出 R$ X 的延迟代价」。**

**应用 —— 以及支撑这个数字的诚实。** 在扼流情景下，**Project J 对他人造成 R$ 3,730 的损害，自己只承受 R$ 853** —— 净额 +2,877：它是**加害者**。**效率高出 30 倍的 Project C，承受 R$ 867 却不造成任何损害** —— 它是**受害者**。净额之和为**零**：每个加害者都对应着一个受害者。

> ⚠️ **但今天池子仍然装得下**（额度的 94%）。**并不存在物理扼流** —— 没人停摆，没人延期。损害是**分配性的**，不是**运营性的**。在池子还有余量时说*「J 在拖慢 C」*，那是**披着严谨外衣的谎言**。因此本模块是**情景化的**，并**明确标注为预测**：它展示池子**从哪一点开始翻车**（消耗 +10% → 整个组合停摆 0.9 天，R$ 1,497），以及**翻车时的代价**。

![定价化的资源争用 —— 谁造成损害、谁承担损害；池子一旦干涸，所有人都停摆，包括毫无过错的高效项目](docs/screenshots/budget-contencao.png)

### 🪓 削减政策 —— 如果组合需要腾出空间，谁该走？

**概念。** 这正是投资组合委员会**永远答不上来**的问题。在有限的池子里，接纳第 N+1 个项目就是**从已在场的 N 个项目那里抽走 token** —— 多接纳一个项目，**所有人被稀释 9.1%**。

**方法论。** 诚实的答案**不是「花得最多的那个」** —— 按毛消耗砍会惩罚一个**又大又能产出**的项目。答案是**「每个 token 交付最少的那个」**：按**效率**（EV ÷ 百万 token）排序，能以**最小的价值代价**释放最多的池子。对角线 `y = x` 把**划算的削减**与**毁掉的比释放的更多的削减**分开。

**应用。** 砍掉 **Project J** 释放**池子的 20.5%**，只牺牲**1.9% 的价值** —— 几乎能腾出 2 个新名额而不稀释任何人。砍掉 **Project F** 只释放 3.4%，却要牺牲 **21.2% 的价值**：那会**毁掉的价值远超它释放的容量**。**这不是「削减成本」—— 这是一个带着数字的、明确的取舍。**

## 🔁 预算再学习闭环 —— 与生态系统**一同进化**的智能体

**健康的生态系统拥有记忆。** 捕食者学会哪种猎物值得追逐；植物学会朝哪个方向生长。没有这种学习就没有适应 —— 只有永无止境的盲目试错。这恰恰是智能体所缺的：它会建议*削减浪费*，却**从不在下一周核查：它自己下令的那次削减是否真的释放了池子。** 只建议而不验证，不是管理 —— 那是循环利用的猜测。

**概念。** PM 智能体闭合了这个环：它在建议削减的那一周**记下数字**，并在下一次周会上**向自己问责**。浪费下降了吗？建议奏效了。没有？它失败了。这与再学习引擎是同一个**上下文赌博机** —— 现在应用到**token**这个维度，也就是生物圈中最稀缺的资源。

**方法论。** 只有它**推荐过**的动作才被评判 —— 它为自己下的指令负责，**绝不把运气释放的池子据为己功**。削减奏效 → 它对该建议的信心**上升**。没奏效 → **下降**。低于 2% 的变动是噪音，而**它不从噪音中学习**。信心是**按项目**的：智能体会摸清哪些削减在*那个*生态系统里真正释放池子。

**直接收益。** 每个周五，智能体带着对自己建议的交代来到周会：*「上周我下令削减 幻觉_代码；释放了 R$ X 的池子 —— 我提高信心」*，或*「没奏效 —— 我降低信心并重新考量」*。久而久之，它不再重复在你的语境里无效的削减，而对有效的加倍下注。**你的预算不再由固定规则来管，而是由一个学会了在你的棋盘上下棋的智能体来管。**

![预算再学习闭环 —— 浪费起起落落，智能体的信心随之回应：削减奏效则上升，失败则下降，忽略噪音](docs/screenshots/budget-loop-reaprendizagem.png)

## 🤖 Project Manager Agent — 读取 10 个维度、持续学习，并且**懂得沉默**

仪表盘负责**诊断**，因果链负责**量化**，而智能体**决定现在该做什么** —— 并且一个周期接一个周期地发现：在*那个具体项目*里，究竟哪根杠杆才真正拨动指针。它扫描**10 个维度**（工期、ROI、风险、Token、成本、模型漂移、可靠性、质量、流动与浪费），把每一个都换算成**等效项目天数 × 该项目自身的延迟成本**，并回答唯一重要的问题：**现在做什么，以及这么做值多少钱。**

> **我们必须修正的弱点。** 智能体**总是**推荐点什么：每个周期它抓住最大的损害然后大喊。**每周都在喊的智能体会变成噪音，而噪音会被忽略** —— 于是它什么也改变不了，无论它多么正确。**它缺的是保持沉默的权利。** 下面三种方法论正是为此而来。

### 🚦 PRINCE2 —— *例外管理*：沉默的权利

**概念。** PRINCE2 的*例外管理*主张：只要项目仍在约定容差之内，就**不应打扰**管理者。当**预测值**突破容差时 —— 不是实际值，而是**预测值** —— 才触发**例外报告（Exception Report）**。

**方法论。** 每个维度一条容差（工期、成本、风险、质量、效益）。升级由**预测值**驱动：蒙特卡洛的 P80 与 EVM 的 EAC。例外报告有四个必备部分 —— **原因、影响、可选方案与建议**。正是*可选方案*这一行，把例外报告与警报区分开来：升级却不提供替代方案，是把问题往上推，而不是管理它。

**在本项目中的应用。** 这些容差**不是我们编造的数字** —— 它们来自项目**已经声明过**的东西：承诺的日期（`prazo_alvo`）、批准的预算（`BAC`）、**风险登记册自身**的分级（`nivel='critico'`），以及**项目自己的质量基线**（对自身的回归，DORA 风格）。只有 ROI 上限是明示的策略 —— 而它就摆在明处，供董事会反对。智能体提供的方案是：**吸收**（烧掉管理储备）、**恢复**（压缩关键路径）或**重新谈判**（挪动日期或削减范围）。

![PRINCE2 容差 —— 每个维度距离例外还有多少余量；只有「质量」突破了容差，也只有它被升级](docs/screenshots/prince2-tolerancias.png)

### 🌡️ CCPM（高德拉特）—— *缓冲管理*与发烧图

**概念。** 在高德拉特的*关键链*中，缓冲不是藏在每个任务里的脂肪，而是**项目末端一块明确的缓冲垫**。**发烧图**把*关键链完成了多少*与*缓冲消耗了多少*交叉起来，告诉你身处三个区域中的哪一个。

**方法论。** 边界是**对角线**的，这正是方法的精髓：在项目**末期**烧缓冲很正常 —— 在**早期**烧掉它则很严重，因为前面还有一整个项目。**绿色 = 什么都别做。黄色 = 规划恢复。红色 = 立刻行动。**

```
verde/amarelo:    y = 1/3 + (1/3)·x
amarelo/vermelho: y = 2/3 + (1/3)·x
```

**在本项目中的应用。** 缓冲就是我们本就在算的**蒙特卡洛进度**里的 `P80 − P50`。消耗量则是把 **Earned Schedule 的偏差**换算成天数。而正是发烧图，给了智能体一个客观的「沉默触发器」：**绿区且在容差内 = 无事可升级。** 今天**10 个项目中有 2 个**正是如此 —— 在没什么可说时保持沉默，智能体才赢得了「有话时被倾听」的权利。

![CCPM 发烧图 —— 10 个项目分布于三个区域；对角线边界使同样的缓冲消耗在末期无害、在早期严重](docs/screenshots/ccpm-fever-chart.png)

### 🏦 PMI —— *储备分析*：应急储备 × 管理储备

**概念。** PMI 区分了几乎所有人都会混淆的两种储备：**应急储备**覆盖*已知的未知*（你**测量过**的变异性），**管理储备**覆盖*未知的未知*（意外冲击）。

**方法论。** `应急储备 = P80 − P50`，`管理储备 = P95 − P80`。再加上几乎没人做的对照：你**拥有的**应急储备，对比你的**风险登记册所证成的**储备（EMV —— *Integrated Cost-Schedule Risk Analysis*，Hulett）。工期缓冲**对风险事件是盲的**；几乎每一份进度计划都在这里发现自己过于乐观。

**在本项目中的应用 —— 以及一堂诚实课。** 把「影响 4」（1–5 量表）换算成天数，需要一个**属于我们、而非属于你**的映射。于是我们**对自己的假设做了压力测试**：把假设的影响砍掉一半，「储备不足」这个结论就从 **10/10 翻转为 1/10 个项目**。这是**刀锋上的结论**，因此它**不会被当作发现来兜售** —— 每个项目都带有 `robusto` 字段，当智能体**自己的解读经不起压力测试时，它会主动提醒你**。而**完全不依赖任何假设**留下的，是纯粹的算术，也是真正的发现：**缓冲仅占关键链的约 9%，而 CCPM 的工作区间是 25–50%。**

### 🏃 Sprint 与周五周会的进度辩论

**概念。** 周五*周会*的进度辩论需要**数字**，而不是意见。意见推不动项目。

**方法论。** 三个指标开启讨论。**（1）Say-do ratio**（`ΔEV ÷ ΔPV`）：say-do 为 0.7 的团队**并不慢** —— 它是在*承诺比自己能交付的多出 30%*。产能不是靠施压修好的；承诺是靠可预测性修好的。而 say-do **远高于 1** 同样不是英雄主义：那是**基线坏了**。**（2）Sprint 的局部 CPI**，**刻意**与累计值分开 —— 累计值是平均数，而平均数会**掩盖**最近那个糟糕的 Sprint：累计 CPI 1.05 完全可能藏着一个 0.60 的末位 Sprint。**局部的在指控；累计的在安慰。** **（3）按速度的预测**：如果团队需要 6 个 Sprint 而只剩 4 个，**日期其实已经死了** —— 却没人察觉，因为累计燃尽图*看起来*仍贴近计划。

**在本项目中的应用。** Sprint **不是虚构的**：它就是 **EVM 的周期**，项目本来就有的节奏，带着真实的 PV/EV/AC。在进度计划之外另建一套 Sprint 日历，等于为同一个项目制造**第二个真相** —— 而两个真相，等同于没有真相。

> **⚠️ 合规性，把话说明白。** 这是一份**以 EVM（ANSI/EIA-748）为基础、采用敏捷启发式指标的节奏化进度报告** —— **它不是 Scrum**。《*Scrum 指南 2020*》**既不包含**「velocity」**也不包含**「burndown chart」（那是市场实践，不是官方工件），并且它已用 **Sprint Goal** 取代了 Sprint Backlog 的 *commitment*，把 backlog 视为**预测**。因此「say-do ratio（已交付 ÷ 已承诺）」属于**行业**词汇，而非正统 Scrum。**指标是诚实的；会撒谎的是标签。**

![Sprint —— 各 Sprint 的 say-do ratio 与实际燃尽对比计划；节奏即 EVM 周期](docs/screenshots/sprints-weekly.png)

### 🎯 雷达与再学习引擎 —— 为什么是这个维度，而不是别的

智能体**不只看赢家** —— 它把整条替补席都摆出来。每个维度化为**等效天数**，天数再按*该项目*的延迟成本化为**金额**，而权重是智能体**在那里学到的**。优先级 = `损害 × 权重`。

**再学习引擎**是一个*上下文赌博机（contextual bandit）* —— 简单、可审计，而且我们把话说明白：**这不是深度学习**。每个周期智能体推荐一根杠杆并**记下它的目标指标**；下个周期它**向自己问责**。变好 → 权重**上升**。变差 → 权重**下降**。低于 2% 的变动是噪音，而**智能体不从噪音中学习**。只有它**推荐过**的那根杠杆会被评判：它为自己下的指令负责，**也绝不把运气带来的改善据为己功**。最终得到的画像**在隔壁项目身上并不适用** —— 而这恰恰就是重点。

![10 个维度的雷达 —— 各维度的损害放在同一把尺子上（R$），以及智能体选择攻击的那一个](docs/screenshots/pm-agent-radar.png)

## 🧰 从零开始的本地安装步骤

> 一切都在**你自己的机器上**运行。包内不含作者的任何密钥，也没有任何数据离开你的电脑。

### 第 0 步 —— 前置条件

| 依赖 | 版本 | 必需？ | 用途 |
|---|---|---|---|
| **Python** | 3.10+ | ✅ | 管线、KPI、蒙特卡洛、多准则决策 |
| **Node.js + npm** | 18+ | ✅ | 仪表盘（Evidence） |
| **git** | 任意 | ✅ | 克隆仓库 |
| **Rust + maturin** | 稳定版 | ⬜ 可选 | 加速日志分类 |
| **tectonic** | 任意 | ⬜ 可选 | 生成 PDF 版 pitch deck |

*在 Windows 上请使用 **WSL** 或 **Git Bash** —— 这条管线是 `bash` 脚本。*

### 第 1 步 —— 克隆仓库
```bash
git clone https://github.com/bpenedo/Gestao-de-Projetos-PM-IA-BSC-DashBoard.git
cd Gestao-de-Projetos-PM-IA-BSC-DashBoard
```

### 第 2 步 —— 独立的 Python 环境
```bash
cd foundations/pipeline
python3 -m venv .venv
source .venv/bin/activate        # Windows (PowerShell): .venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 第 3 步 —— 安装仪表盘依赖
```bash
cd ../evidence
npm install
```

### 第 4 步 —— 运行演示（匿名，无需凭证）
```bash
cd ../pipeline
./run_all.sh --mock
```

管线依次执行：匿名演示数据 → KPI → NPV/IRR/MIRR/EAA/PI → **将分布拟合到 token** → **蒙特卡洛（10,000 次迭代）** →
AHP-TOPSIS 2n → **DEMATEL · ELECTRE · PROMETHEE · MAUT · MCDA-C** → **排名稳健性（Dirichlet）** → 图表 → 管理档案 →
5D 地图 → pitch deck → 构建仪表盘。

### 第 5 步 —— 打开仪表盘
```bash
cd ../evidence
npm run dev          # http://localhost:3000
npm run preview      # (alternativa) serve o estático já compilado em build/
```

### 第 6 步 —— 切换到你自己的数据

**6.1 —— 凭证与参数**（全部可选；没有 `.env` 时管线使用默认值）：
```bash
cd ../pipeline
cp .env.example .env      # edite: LANGFUSE_PUBLIC_KEY, LANGFUSE_SECRET_KEY, SELIC_ANUAL, USD_BRL...
```

**6.2 —— 你的现金流**（它驱动 NPV、IRR 和蒙特卡洛）：
```bash
cp fluxo_caixa_template.csv fluxo_caixa.csv
```
格式：`periodo 0` 是投资（负现金流），`taxa` 是每期折现率（`0.10` = 10%）。
```csv
project_name,periodo,fluxo,taxa
Project A,0,-12000,0.10
Project A,1,3000,0.10
Project A,2,4000,0.10
```

**6.3 —— 用真实数据运行：**
```bash
./run_all.sh          # sem --mock: sincroniza do Langfuse e usa fluxo_caixa.csv
```

### 第 7 步（可选）—— 加速与 PDF
```bash
cd analise_rs && maturin develop --release && cd ..   # Rust (PyO3): classificação mais rápida
```
若要生成 pitch deck，请安装 **tectonic**（例如 `cargo install tectonic` 或用你的发行版包管理器）。

### 🩺 常见问题

| 现象 | 可能原因 | 解决 |
|---|---|---|
| `no such table: ...` | 数据库未初始化 | `python3 db.py` |
| 仪表盘构建失败 | 旧的构建产物 | `rm -rf ../evidence/build && npm run build` |
| `findfont: Failed to find font weight` | matplotlib 警告 | 无害，可忽略 |
