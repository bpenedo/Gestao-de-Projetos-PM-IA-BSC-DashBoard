### 💸 당신은 매달 AI에 돈을 냅니다. 그런데 AI는 **당신에게** 돈을 돌려주고 있나요?

**ChatGPT, Claude, Copilot, Gemini, Perplexity, DeepSeek, Kimi, Qwen…**가 카드를 긁을 때마다 **수백만 달러짜리** 질문이
답을 얻지 못합니다: **수익은 어디 있나요?** 실제로 몇 인시(人時)를 절약했나요? 당신의 돈은 환각·재작업·대기 속에서 얼마나
**증발**했나요? 어떤 AI 프로젝트가 **오늘 확장할 자격**이 있고, 어떤 프로젝트가 "혁신"이라는 박수 속에서 **현금을 흘리고**
있나요?

당신에게는 'AI 비용'이 있는 게 아니라 **조용한 누수**가 있습니다 — 그것도 눈을 가린 채로. *"측정되지 않는 것은 관리할 수도,
개선할 수도 없다"* — 그리고 시장은 당신 대신 측정하고, 당신에게 청구서를 보냅니다.

**이 프레임워크는 불을 켭니다.** AI 구독의 **보이지 않는 지출**을 **측정 가능하고, 비교 가능하며, 감사 가능한 수익**으로
바꿉니다 — **균형성과표**(Kaplan & Norton), **월스트리트급 투자 분석**, **다기준 의사결정**의 엄밀함으로. 이것이 *바라는 것*과
*아는 것*의 차이, AI에 **돈을 내는 것**과 AI로 **돈을 버는 것**의 차이입니다.

> *"측정되지 않는 것은 관리할 수도, 개선할 수도 없다."* — Kaplan & Norton

> *"정밀하게 측정하는 자가 탁월하게 짓는다."* — Pierre Vernier

> *기도하고 공부할 때, [나의 말]이 너를 떠나지 않게 하라. 네 입술에서 나오는 모든 단어와 표현마다, 하나됨(Unification)을 이루려는 마음을 품으라.* — Aryeh Kaplan

> *순수 형이상학은 그 본질상 모든 형태와 모든 우연 위에, 그 너머에 위치하여 동양의 것도 서양의 것도 아니다: 그것은 보편적이다.* — René Guénon

> *자기 자신을 아는 것은 자신의 혈통과 자신의 힘을 아는 것이다.* — Harvey Spencer Lewis

> *Scientia es Lux Lucis* ∞ Sapere Aude S∴A∴☬ ☿

> 🐺 **어둠 속에서 AI에 돈 내는 것을 멈추세요.** 시장이 믿음으로 AI를 구독하며 **Gartner** 통계(파일럿 이후 폐기되는 GenAI
> 프로젝트 ≥30%)가 되는 동안, **당신**은 모든 토큰을 측정하고, 승자 프로젝트를 선정하며, 보이지 않는 지출을 **감사 가능한

> 당신의 것입니다. 이제 그것을 수익성 있게 만드세요** — 무료로, 당신의 컴퓨터에서, **5분** 만에:

## 🌅 왜 이것이 판을 바꾸는가

**AI 시대에는 두 종류의 사람이 있습니다.** 첫 번째는 모든 것을 구독하고, 크게 지출하며, 잘되기를 **기도**합니다 — 그리고 파일럿에서
죽는 프로젝트라는 잔인한 통계를 부풀립니다. 두 번째는 월스트리트가 진지한 자산에 하는 일을 합니다: **측정하고, 비교하고,
우선순위를 매기고, 재배분**합니다 — 그리고 모든 구독 달러를 **복리 수익**으로 바꿉니다. 둘 사이의 유일한 차이는 **재능도 예산도
아닌, '계기화(instrumentation)'**입니다.

생성형 AI는 새로운 종류의 반복 지출 — **구독과 토큰** — 을 만들었고, 그와 함께 이 시대 가장 값비싼 낭비를 가져왔습니다:
**보이지 않는 낭비.** 보이지 않는 것은 고칠 수 없고, 측정하지 않는 것은 확장할 수 없으며, 증명하지 못하는 것은 이사회가 승인하지
않습니다.

**이 프로젝트는 당신을 첫 번째 부족에서 두 번째 부족으로 옮깁니다.** 모든 AI 프로젝트를 **금융 자산**으로 계기화하고
**균형성과표**, **투자 분석(NPV, IRR, MIRR, EAA, PI, 회수기간)**, **린 식스 시그마** 아래에서 측정합니다 — 나아가 다기준
모델(**AHP-TOPSIS 2n**)로 **당신 포트폴리오에서 최고의 프로젝트를 선정**합니다. 불투명한 월 청구서가 **감사 가능한 투자
논거**가 됩니다: 어디를 확장할지, 어디를 끊을지, 구독이 어디서 **몇 주** 만에 회수되는지 — 그리고 어디서 무방비로 출혈하는지를
숫자로 알게 됩니다.

우리는 새로운 영토의 **개척자**입니다 — **인공지능과 고부가가치 금융 지능 사이의 경계**. 아직 지도에 없는 땅을 측량하는 탐험가처럼, 이
프레임워크는 구독의 안개를 **명확한 수익 항로**로 바꾸는 **나침반**(🧭)입니다: 토큰 하나하나가 1마일, 프로젝트 하나하나가 이익을
향한 원정입니다. 맹목적 비용이 있던 곳에 **측정 가능한 기회**가 태어나고, 죽은 스프레드시트가 있던 곳에 **살아있는 투자 논거**가
피어납니다.

### 🚀 마이크로 SaaS, SaaS, 스타트업, 솔로프리너를 위해

제품에 AI를 넣는 순간 아무도 말해주지 않은 사실: **당신은 방금 AI를 마케팅 예산에서 COGS(매출원가)로 옮겼습니다.**
그리고 사용량에 따라 커지는 COGS는 비용이 아닙니다 — **총이익률에 잡힌 담보 대출**입니다. 신규 사용자 한 명마다
토큰이 듭니다. 환각으로 인한 재시도 한 번마다 마진이 두 번 탑니다. 그리고 청구서는 월말에야 나타납니다, 되돌릴 수
없을 때.

| 당신이… | 아무도 측정하지 않는 고통 | 이 프레임워크가 돌려주는 것 |
|---|---|---|
| **솔로프리너** | 당신이 *곧* 팀이며, 당신의 한 시간이 가장 비싼 자산 | **토네이도**가 결과를 움직이는 변수를 짚어 줍니다 — 즉 **다음 한 시간을 어디에 투자할지** |
| **마이크로 SaaS** | 토큰 비용이 사용량과 함께 커지며 조용히 마진을 갉아먹음 | **당신의 실제 토큰에 적합된** 분포 + **CVaR**: 나쁜 달의 가격이 도착 *전에* 매겨짐 |
| **규모화된 SaaS** | 모든 AI 기능이 같은 로드맵을 놓고 싸우는 프로젝트 | **다섯 방법**이 무엇을 출시할지 정하고, **견고성**이 1위가 가중치 2%p 오차를 견디는지 알려줌 |
| **투자 유치 중인 스타트업** | 투자자는 *"우리는 AI를 씁니다"*를 사지 않음 | 그들이 사는 것은 **NPV, IRR, 회수기간, `P(NPV<0)`** — 그리고 **피치 덱이 LaTeX로 바로 나옵니다** |

**실제로 달라지는 것.** 당신의 총이익률은 추정치이길 멈추고 **꼬리에 가격이 매겨진 분포**가 됩니다. 런웨이는 단순
나눗셈이길 멈추고 **VaR**를 얻습니다: *"스무 개 시나리오 중 열아홉에서, 내 현금은 최소 N개월 버틴다."* 그리고 보드
덱이나 실사의 순간, 당신은 아무도 재현 못 하는 스프레드시트를 여는 게 아니라 **고정 시드**를 가진 숫자를 엽니다.
어떤 공동창업자도, 투자자도, 감사인도 다시 돌려 **정확히 같은** 값을 얻습니다.

> **잔인한 재포지셔닝:** 솔로프리너가 CFO처럼 결정하기 시작합니다. 그리고 CFO가 솔로프리너의 속도로 결정하기
> 시작합니다.

> **약속:** *AI에 돈 내는 사람*을 *AI로 돈 버는 사람*으로 — *AI를 쓰는 사람*을 그것을 **개척적으로 통달하고, 측정하고,
> 배가하는 사람**으로 바꿉니다. 믿음이 아니라 숫자로.

## 📈 증거 (Gartner · IDC · PwC · McKinsey · MIT)

제 말을 믿지 마세요. **수십 년간 이를 연구해 온 기관들을 믿으세요** — 그 평결은 만장일치입니다: **AI는 막대한 가치를 창출하지만,
측정하고 지배하는 자에게만 전달한다.** "통달 없이 AI를 쓰는" 사람은 폐기 통계가 되고, 수익을 계기화하는 사람은 **상을 차지합니다.**

- 🧭 **Gartner** — **2025년 말까지 생성형 AI 프로젝트의 ≥ 30%가 개념 증명 이후 폐기**될 것이며, 핵심 원인은 **불분명한 비즈니스
  가치**(그 외 열악한 데이터, 치솟는 비용, 취약한 통제)라고 예측했습니다. *→ 측정 없이는 프로젝트가 파일럿에서 죽는다.*
- 🔬 **MIT** (보고서 *"The GenAI Divide / State of AI in Business 2025"*, NANDA 이니셔티브) — **기업 GenAI 파일럿의 절대
  다수가 손익(P&L)에 측정 가능한 영향을 주지 못한다**고 널리 보도되었습니다; 가치를 전달하는 소수는 AI를 **프로세스와 측정**과
  결합합니다. *→ 차이는 채택이 아니라 측정이다.*
- 💵 **IDC** (연구 *"The Business Opportunity of AI"*, Microsoft 후원) — **측정하고 최적화하는** 조직은 AI에 투자한 **1달러당
  여러 달러** 규모의 수익을 보고했으며, 선도자와 후발자 간 편차가 큽니다. *→ ROI는 존재한다 — 그리고 계기화하는 자를 편애한다.*
- 🌍 **PwC** (*"Sizing the Prize"*) — AI가 2030년까지 세계 경제에 최대 **~15.7조 달러**를 더할 수 있다고 추정합니다; 그러나
  상은 가치를 **포착**하는 자의 것이지 단지 소비하는 자의 것이 아닙니다. *→ 파이는 거대하다; 그 조각은 측정하는 자의 것이다.*
- 🏆 **McKinsey** (*"The State of AI"*)와 **BCG × MIT Sloan** — 소수의 **"AI high performers"** 집단이 불균형적인 수익을
  포착합니다; 전환점은 AI를 수익이 입증된 곳의 **지표·거버넌스·재투자**와 결합할 때 옵니다. *→ 승자는 측정하고, 우선순위를 매기고,
  재배분한다.*

> **이 프레임워크가 건너는 것이 바로 그 간극입니다:** 그것은 당신을 *파일럿에서 포기하는* 쪽에서 **실제로, 입증된 결과**를 가진
> 쪽으로 옮깁니다 — BSC, 투자 분석, 다기준 의사결정으로.

> ⚠️ **정직성 참고(꼭 읽어 주세요):** 위 수치는 이 기관들의 실제 헤드라인을 반영하지만 **보고서와 백분율은 갱신됩니다** — 공식
> 자료에 인용하기 전에 **1차 출처**(Gartner Newsroom; IDC/Microsoft *Business Opportunity of AI*; PwC *Sizing the Prize*;
> McKinsey *State of AI*; MIT *State of AI in Business*)에서 정확한 값과 연도를 확인하세요. 여기서는 **방향성 있는 근거**로
> 쓰였을 뿐, 숫자 보증이 아닙니다.

## 💥 아무것도 하지 않는 것의 비용 (아무도 하지 않는 계산을 해보라)

**AI PRO** 구독은 좌석당 월 **20~200달러**입니다. 팀 인원 수를 곱하세요. 12개월을 곱하세요. 이제 기관들이 **이미 입증한** 것을
적용하세요: **Gartner**는 **≥ 30% 폐기**를 전망하고, **MIT**는 **대다수 파일럿이 수익을 내지 못한다**는 것을 보여줍니다. 이
총액의 큰 부분은 투자가 아니라 **순수한 출혈**입니다.

> **직접적 예시(당신 숫자로 바꿔 보세요):** 10좌석 × 월 30달러 × 12 = **연 3,600달러**. 그중 ~30%가 보이지 않는 낭비가 되면
> **연 ~1,080달러가 증발**합니다 — 작은 팀 하나에서, 단 1년에. 당신의 실제 숫자라면 충격은 더 큽니다.

그리고 아픈 부분은 이것입니다: **이 비용은 복리로 불어나고, 기다려 주지 않습니다.** 측정 없는 매달은 **돌아오지 않는** 누수의
달입니다 — 그동안 계기화한 경쟁자는 이미 **수익 나는 곳으로 자본을 재배분**하고 있습니다. 선점 우위는 일찍 만들어집니다:
**먼저 측정하는 자가 먼저 확장한다.**

시작하기에 가장 비용이 낮은 순간은 어제였습니다. 두 번째로 좋은 때는 **지금**입니다 — 그리고 그 비용은 **0달러**와 **5분**입니다.
질문은 *"측정할 여유가 있는가?"*가 아니라 ***"'측정하지 않을' 여유가 얼마나 더 있는가?"***입니다.

## 🎲 몬테카를로 — 평균이 숨기는 리스크

**평균적으로** 양(+)인 NPV는 누구도 지켜주지 않습니다. 평균은 금융에서 가장 안락한 거짓말입니다: 결코 오지 않을 수도 있는
시나리오를 묘사할 뿐입니다. 당신의 운명을 정하는 것은 **꼬리** — 나쁜 날입니다.

이 프레임워크는 프로젝트마다 **10,000개의 미래**를 시뮬레이션합니다. 모든 현금흐름이 **확률변수**가 되고, 포트폴리오 전체가 반복마다 다시 계산됩니다. 끝에 남는 것은 하나의 숫자가 아니라
**당신 돈의 분포 전체**입니다:

- **`P(NPV < 0)`** — 손실의 실제 확률. 아무도 보여주지 않는 그 숫자.
- **VaR 5%** — 가장 나쁜 그럴듯한 시나리오: *"스무 개의 미래 중 열아홉에서, 나는 최소 이만큼 번다."*
- **CVaR 5%** — 재앙이 실제로 닥쳤을 때, 평균적으로 얼마의 대가를 치르는가.
- **민감도 토네이도** — 다중회귀와 피어슨 상관: 어떤 변수가 실제로 NPV를 움직이는가.
- **20가지 입력 분포**, 검증된 **상관행렬**(주변분포를 정확히 보존하는 Iman-Conover), **1%~99% 백분위수**,
  그리고 100구간 히스토그램.

고정 시드: 다시 돌리면 **정확히** 같은 결과. 감사 가능 — "마법"이 아닙니다.

> **전환점:** 당신은 NPV가 가장 큰 프로젝트를 고르는 대신 **나쁜 시나리오에서 살아남는 프로젝트**를 고르게 됩니다.
> 그것이 리스크 관리 — 투자자와 도박꾼을 가르는 선입니다.

멈추고 **엔지니어링**이 되는 순간입니다.

## 🔬 신호는 상류에 있다 — 레버리지도 거기에 있다

프레임워크 자체를 측정하다가 발견했습니다: NPV의 민감도 토네이도가 **정확히**
`1.0 · 0.9091 · 0.8264 · 0.7513…` 을 돌려주었습니다 — 할인계수 `1/(1+i)ᵗ` 입니다. NPV는 현금흐름에 대해 **선형**이므로,
현금흐름만 시뮬레이션해서는 할인율 외에 아무것도 알 수 없습니다. **진짜 확률적 신호는 상류, 즉 토큰에 있습니다.**

### 1️⃣ 분포를 임의로 정하지 마십시오. 데이터에 맞추십시오.

열한 개의 후보 분포를 실제 토큰 소비 시계열(`logs_langfuse`)에 **최대우도**로 적합시킵니다. **AIC가 가장 낮은** 것이
이깁니다 — AIC는 추가 모수마다 벌점을 주어 과적합을 막습니다 — 그리고 **콜모고로프-스미르노프** 검정이 적합도를 잽니다.
이것이 고전적인 *데이터에 분포 적합*이며, 소비의 **두꺼운 꼬리**를 드러냅니다: 어떤 프롬프트는 전형값의 10배를
쓰고, 예산을 터뜨리는 것은 바로 그 꼬리입니다 — 평균만 보는 사람에게는 보이지 않습니다.

**그리고 적합이 나쁘면 프레임워크는 소리칩니다.** KS의 p-값이 0.05 아래로 떨어지면 화면은 정밀함을 가장하는 대신
`적합도 취약`을 붉은 글씨로 경고합니다. 예쁜 숫자보다 정직한 숫자가 낫습니다.

![실제 토큰에 분포 적합 — 11개 후보, AIC 선택, 콜모고로프-스미르노프 적합도](docs/screenshots/ajuste-distribuicoes.png)

### 2️⃣ 가중치가 2%p 틀려도 당신의 순위는 살아남습니까?

모든 다기준 방법은 **암묵적 100% 확신**과 함께 승자를 내놓습니다. 그러나 기준 가중치는 추정치이지 계시된 진리가 아닙니다.
IITA 가중치의 2%p가 1위와 2위를 뒤바꾼다면, 그 "승자"는 보정의 산물일 뿐입니다.

그래서 DEMATEL 가중치를 **디리클레**로 섭동합니다 — `w' ~ Dir(κ·w)`, 정확히 단체(simplex) 위에 살며 `E[w'] = w`를
보존하므로 **편향 없이** 흔듭니다 — 그리고 **2,000번** 다시 순위를 매깁니다. 평결의 성격이 바뀝니다:

> *"Project C가 최고다"* ⟶ **"Project C는 그럴듯한 선호 우주의 99.9%에서 승리한다"**

이는 **의사결정 그 자체에 대한 신뢰구간**입니다. 그리고 합의가 숨기고 있던 것을 드러냅니다: 아래 화면에서
**PROMETHEE II는 우주의 25.4%에서만 선두를 선택합니다**. 네 학파는 동의하고, 하나는 정면으로 반대합니다. 이는 잡음이
아니라 경고입니다: 그 선택은 당신이 *우월관계*를 *효용*보다 선호하는지에 달려 있습니다. 어떤 단일 순위도 그것을 알려주지
않습니다.

![디리클레 섭동에 의한 순위 견고성 — 승리 확률과 학파 간 불일치](docs/screenshots/robustez-dirichlet.png)

### ⚡ 구체적인 레버리지

| 자원 | 이전 | 이후 |
|---|---|---|
| **시간** | 어떤 프로젝트를 확장할지 몇 주간 논쟁 | 평결이 확률과 함께 도착 — 논쟁은 회의 한 번으로 끝남 |

| **평판** | *"내 생각엔 이게 최고다"* | *"99.9%의 시나리오에서 이깁니다; 반대하는 방법은 이것이고, 이유는 이렇습니다"* |

### 💼 월 20달러 요금제에서 20만 달러 계약까지

**프리랜서나 중소기업이라면:** 적합된 분포는 나쁜 토큰의 달이 오기 전에 **그것이 얼마를 쓸지** 알려줍니다 — 그리고
견고성은 정말로 다른 프로젝트로 노력을 옮길 가치가 있는지, 아니면 둘이 오차 범위 안에서 동률인지 알려줍니다. 빠듯한
현금으로 어둠 속에서 최적화하는 일이 끝납니다.

**대기업이라면:** `P(승리)`는 투자위원회에 빠져 있던 조각입니다. *"A팀이 프로젝트 X를 민다"*를 **"프로젝트 X는 방어
가능한 가중치 보정의 99.9%에서 이기며, 유일하게 반대하는 학파는 우월관계이고 근거는 기준 Y다"**로 바꿉니다. 정치적 논쟁이
**기술적 논쟁**이 되고 — CFO는 감사에서 살아남는 숫자를 얻습니다.

## 🎓 기초: 몬테카를로란 무엇이며, 다기준 의사결정이란 무엇인가

### 🎲 몬테카를로 수학적 시뮬레이션

#### 📖 개념

몬테카를로는 불확실한 시스템에 대한 질문에 **무작위 추출로** 답하는 수치적 방법입니다. 그 발상은 수학자의 본능을 뒤집습니다:
**닫힌 형태**로 답을 유도하는 대신 — 적분을, 조합론을, 미분방정식을 푸는 대신 — **모형**을 세우고, 불확실한 변수들의 실현값을
수천 번 추출하고, 그저 **무슨 일이 일어났는지 세는** 것입니다. 끝에 남는 것은 하나의 숫자가 아니라 **결과의 확률분포**입니다.

두 법칙이 이를 떠받칩니다. **큰 수의 법칙**은 시뮬레이션 평균이 참값으로 수렴함을 보장합니다. **중심극한정리**는 그 속도를
알려줍니다: 표준오차는 `1/√N`로 줄어듭니다. 그 결과는 정직하고 다소 잔인합니다 — **정밀도를 두 배로 하려면 반복을 네 배로
해야 합니다**. 몬테카를로는 빠르지 않습니다. 그 미덕은 다른 데 있습니다: 오차가 **문제의 차원에 의존하지 않는다**는 것.
결정론적 구적법은 *차원의 저주*에 시달려 변수 수십 개에서 무너지지만, 몬테카를로는 그렇지 않습니다. 해석적 수학이 죽는 바로 그
자리에서 이깁니다.

#### 📖 언제 어디서 태어났는가

**1946년, 뉴멕시코주 로스앨러모스.** 폴란드 수학자 **스타니스와프 울람(Stanisław Ulam)**은 뇌염에서 회복하며 며칠이고 솔리테어를
했습니다. 그는 한 판을 이길 확률이 얼마인지 궁금해졌습니다. 조합론으로 풀어보려다 포기했습니다: 다룰 수 없었습니다. 그때
반칙처럼 보일 만큼 단순한 생각이 떠올랐습니다 — **백 판을 해보고, 이긴 판을 세고, 나눈다**. 그는 즉시 깨달았습니다. 이것은 카드
놀이의 요령이 아니라, 아무도 풀 줄 모르는 문제들을 위한 **일반적인 적분법**이라는 것을.

그는 이 발상을 **존 폰 노이만(John von Neumann)**에게 가져갔고, 폰 노이만은 곧바로 맨해튼 프로젝트에서 그들을 붙들고 있던 문제,
곧 핵분열 물질 속 **중성자 확산**에의 적용을 알아보았습니다. 수천 개 중성자의 무작위 행보 — 산란, 흡수, 핵분열 — 를
**시뮬레이션**하는 일은 가능했지만, 수송 방정식을 푸는 일은 불가능했습니다. **니컬러스 메트로폴리스(Nicholas Metropolis)**는
울람의 삼촌이 돈을 빌려 도박하던 모나코의 카지노를 따 **"몬테카를로"**라는 이름을 제안했습니다. **ENIAC**이 최초의 계산을
가능하게 했고, **1949년** 메트로폴리스와 울람은 *"The Monte Carlo Method"*를 *Journal of the American Statistical Association*에
발표했습니다.

이 방법은 문자 그대로 **카드 게임**과 **원자폭탄**의 만남에서 태어났습니다. 이토록 당혹스러운 출생증명서를 가진 과학적 착상은
드뭅니다.

#### 📖 방법론, 다섯 단계

1. **모형화.** 출력을 입력의 함수로 씁니다: `y = f(x₁, …, xₙ)`.
2. **분포 부여.** 불확실한 입력마다 분포를 줍니다. **과거 데이터**가 있으면 거기에 *적합*시키고, 없으면 지정합니다 — 그리고 그
   사실을 **명시해야** 합니다.
3. **추출.** `N`개의 시나리오를 뽑습니다. 변수들이 **상관**되어 있다면 추출은 그 구조를 존중해야 합니다: 실제 종속성이 있는 곳에서
   독립적으로 뽑는 것이 이 방법의 가장 흔한 오류입니다.
4. **전파.** 각 시나리오에서 `f`를 계산합니다. 바로 여기서 입력의 불확실성이 출력의 불확실성으로 **변환**됩니다. 선형 근사는
   전혀 없습니다.
5. **분석.** 얻어진 분포를 살핍니다: 평균과 표준편차, **백분위수**, 사건 확률(`P(y < 0)`), **꼬리**(VaR, CVaR), 그리고
   **민감도**(어떤 `xᵢ`가 `y`를 움직이는가).

#### 📖 세계에서의 용도와 응용

- **금융:** 이색 옵션 가격결정(닫힌 형태가 없는 경우), 포트폴리오의 **VaR**와 **CVaR**, 규제 스트레스 테스트(바젤), 신용리스크.
- **공학:** 구조 신뢰성, 제조 공차, 복잡계의 고장 분석.
- **프로젝트 관리:** 일정·비용 리스크(PERT의 확률론적 진화), 완료 S-곡선.
- **물리·화학:** 입자 수송, 방사선 차폐, 통계역학.
- **운영과 공급망:** 대기행렬, 재고, 불확실한 수요 하의 생산능력.
- **역학:** 질병 확산과 불확실성 하의 정책 평가.
- **AI 자체 안에서:** **MCMC**(베이지안 추론), **MCTS** — 알파고가 이세돌을 넘게 한 트리 탐색 — 그리고 신경망의 불확실성을
  추정하는 *몬테카를로 드롭아웃*.

#### 🔒 이 프로젝트에서만의 방법론, 사용, 적용

여기서 몬테카를로는 학술적 장식이 아닙니다: 포트폴리오의 **리스크 엔진**입니다.

- **입력.** 각 기간별 현금흐름은 **삼각분포** 변수(`최솟값`, `최빈값`, `최댓값`)가 되며, 최빈값은 결정론적 값이고 양 꼬리는
  ±30%입니다. 반면 **토큰 소비** — 유일하게 진짜 두꺼운 꼬리를 가진 변수 — 는 **임의로 지정되지 않습니다**: 열한 개의 후보
  분포가 당신의 실제 `logs_langfuse` 시계열에 **최대우도**로 적합되고, **AIC가 가장 낮은** 것이 이기며, 적합도는
  **콜모고로프-스미르노프**로 측정됩니다. p-값이 0.05 아래로 떨어지면 화면은 정밀함을 가장하는 대신 붉은 글씨로 `적합도 취약`을
  찍습니다.
- **상관.** 현금흐름이 종속적일 때, 추출은 **Iman-Conover**를 씁니다. 이는 순위 상관을 부여하면서 **주변분포를 정확히
  보존**합니다. 행렬은 사전에 검증됩니다: 대칭, 대각 1, 양정치.
- **전파.** 프로젝트당 **10,000회 반복**, **고정 시드(42)**: 다시 돌리면 정확히 같은 숫자가 나옵니다. 이것은 사소한 사항이
  아니라, 결과를 동업자·투자자·감사인이 **감사 가능**하게 만드는 것입니다.
- **출력.** NPV만이 아닙니다: **NPV, IRR, MIRR, EAA, PI**와 **토큰 비용**을 시뮬레이션하며, 각각에 대해 열 개의 고전적 기술
  통계량(왜도와 첨도는 Excel 정의), **1%~99% 백분위수**, **100구간 히스토그램**을 제공합니다.
- **정작 중요한 리스크.** `P(NPV < 0)`은 손실의 실제 확률입니다. **VaR 5%**는 가장 나쁜 그럴듯한 시나리오입니다 —
  *"스무 개의 미래 중 열아홉에서 나는 최소 이만큼 번다"*. **CVaR 5%**는 아무도 묻지 않는 것에 답합니다: 재앙이 닥쳤을 때
  **평균적으로 얼마의 비용이 드는가**.
- **민감도.** **토네이도**는 두 가지 고전적 형태로 계산됩니다: **다중회귀의 베타**(어떤 입력이 +1 늘 때 NPV에 미치는 효과)와
  **피어슨 상관**(그 입력의 불확실성이 NPV의 불확실성을 얼마나 좌우하는가). 상호보완적 독법이며, 대시보드는 둘 다 보여줍니다.
- **프레임워크가 스스로에 대해 한 발견.** 자기 자신을 측정하자 토네이도는 **할인계수와 정확히 같은** 베타 `1/(1+i)ᵗ`를
  돌려주었습니다 — NPV가 현금흐름에 대해 *선형*이기 때문입니다. 따라서 현금흐름만 시뮬레이션해서는 할인율 외에 아무것도 알 수
  없습니다. **진짜 확률적 신호는 상류, 토큰에 있습니다.** 이 발견이 실제 데이터에 대한 분포 적합을 촉발했습니다.
- **리스크가 결정을 먹인다.** 몬테카를로의 두 출력이 다기준 모형에 **기준**으로 들어갑니다: `P(NPV<0)`은 **비용** 기준으로,
  **VaR 5%**는 **편익** 기준으로. 따라서 최종 선택은 기댓값만이 아니라 처음부터 **리스크 조정**되어 태어납니다.


**무엇인가.** 어려운 질문에 **제비뽑기로** 답하는 방법입니다. 불확실한 시스템의 수학을 닫힌 형태로 푸는 대신 — 대개
불가능합니다 — 입력 변수에 **확률분포**를 부여하고, 수천 개의 시나리오를 추출하고, 각각의 결과를 계산한 뒤 출력의
**분포 전체**를 관찰합니다. 큰 수의 법칙이 수렴을 보장하며, 오차는 `1/√N`로 줄어듭니다. 즉 **반복을 네 배로 늘리면
오차는 절반**이 됩니다.

**어떻게 태어났는가.** 로스앨러모스, 1946년. 병에서 회복 중이던 **스타니스와프 울람(Stanisław Ulam)**은 솔리테어를
하다가 이길 확률이 얼마일지 궁금해졌습니다. 조합론을 푸는 일은 잔혹하지만, 수백 판을 **시뮬레이션**해서 세어보는 일은
사소하다는 것을 깨달았습니다. 그는 이 착상을 **존 폰 노이만(John von Neumann)**에게 가져갔고, 두 사람은 맨해튼
프로젝트에서 그들을 붙들고 있던 문제 — 핵분열 물질 속 **중성자 확산** — 에 적용했습니다. **니컬러스
메트로폴리스(Nicholas Metropolis)**가 울람의 삼촌이 도박하던 모나코의 카지노를 따 "몬테카를로"라 이름 붙였습니다.
**ENIAC**이 최초의 계산을 가능케 했습니다. 이 방법은 문자 그대로 카드 게임과 원자폭탄의 만남에서 태어났습니다.

**오늘날 어디에 쓰이는가.** 금융의 옵션 가격결정과 **VaR**; 공학의 구조 신뢰성; 프로젝트 관리의 일정·비용 리스크;
입자물리; 공급망; 역학. 그리고 AI 자체 안에서 **MCMC**(베이지안 추론)와 **MCTS** — 알파고가 이세돌을 넘게 한 트리
탐색.

**여기서 우리를 어떻게 돕는가.** 프로젝트의 모든 현금흐름이 확률변수가 되고, 토큰 소비는 **당신의 실제 데이터에 적합된**
분포를 받습니다. 10,000개의 시나리오를 돌리면, 끝에 남는 것은 하나의 NPV가 아니라 **당신 돈의 분포**입니다:
`P(NPV < 0)`(실제 손실 확률), **VaR 5%**(가장 나쁜 그럴듯한 시나리오), **CVaR 5%**(재앙이 닥쳤을 때의 비용), 그리고
**토네이도**(어떤 변수가 실제로 결과를 움직이는가). 평균은 거짓말하고, 꼬리가 결정합니다.

### 🧮 다기준 의사결정 분석(MCDA)

#### 📖 개념과 그 쓸모

프로젝트 사이에서 고르는 일이 어려운 데에는 어떤 스프레드시트로도 풀리지 않는 두 가지 이유가 있습니다. 첫째, 기준들이
**충돌**합니다: NPV가 가장 높은 프로젝트가 대개 가장 위험합니다. 둘째, 기준들은 **통약 불가능**합니다: 원화와 환각 비율과
재작업 시간을 더하는 정직한 산술은 존재하지 않습니다.

MCDA(*Multi-Criteria Decision Analysis*, 다기준 의사결정 분석)는 바로 이 문제와 맞서는 분야입니다 — 1960~70년대, 경영과학과
의사결정 이론의 경계에서 태어났습니다. 이 분야는 **정답을 약속하지 않습니다.** 더 쓸모 있는 것을 약속합니다: 그 선택을
**명시적이고, 감사 가능하며, 방어 가능하게** 만드는 것.

그 창립 명제는 불편하면서 동시에 해방적입니다: **진공 속의 "최선"은 없다.** *누군가가 명시해 놓은 선호 체계 아래에서의* 최선이
있을 뿐입니다. 모든 의사결정자는 이미 어떤 선호 체계로 작동하고 있습니다 — 차이는, MCDA가 없으면 그것이 **암묵적이고,
비일관적이며, 감사 불가능**하다는 점입니다. 암묵적 의견을 명시적 모델로 바꾸는 것 — 그것이 이득의 전부입니다.

#### 📖 세계에서의 용도와 응용

공급업체 선정과 포트폴리오 우선순위; 에너지 기술 선택(태양광 × 풍력 × 바이오매스); 공장·병원·매립지 입지; 환경영향평가;
공공정책과 예산 배분; 인력 선발; 유지보수 우선순위; 그리고 점점 더 많아지는 신흥 기술의 **기술경제성 평가** — AI 프로젝트
포트폴리오가 바로 이 경우입니다.

#### 📖 세 개의 의사결정 학파

- **미국 학파(가치와 효용).** 모든 것을 **하나의 숫자**로 집계합니다. 한 기준의 나쁜 점수가 다른 기준의 뛰어난 점수로
  **보상**될 수 있다고 가정합니다. 단순하고 강력하며 — 때로는 위험합니다. `AHP`, `MAUT`.
- **유럽 학파(우월관계).** **베르나르 루아(Bernard Roy)**가 창시했습니다. 두 대안이 **비교 불가능**할 수 있음을 받아들이고
  **거부권**을 인정합니다: 한 기준에서의 파국적 성과는 다른 곳의 탁월함으로 **매수되지 않습니다**. 의사결정자의 실제 망설임을
  **임계값**으로 모형화합니다. `ELECTRE`, `PROMETHEE`.
- **구성주의 학파.** 모델은 *발견*되는 것이 아니라, 문제의 구조화와 참조 수준에 정박된 척도를 통해 **의사결정자와 함께
  지어지는** 것입니다. `MCDA-C`.

#### 📖 1. DEMATEL — *Decision-Making Trial and Evaluation Laboratory*

**무엇인가.** **Gabus & Fontela**가 **Battelle 기념 연구소**(제네바, 1972-73)에서 복잡하게 얽힌 세계 문제를 연구하기 위해
만들었습니다. 대안을 **순위 매기지 않습니다**: **기준들 사이의 인과 구조**를 지도화합니다.

**어떻게 작동하는가.** 전문가들이 **직접 관계 행렬** `Z`를 채웁니다(기준 *i*가 *j*에 미치는 영향, 0~4). `s = max(최대 행 합,
최대 열 합)`으로 정규화하면 **총 관계 행렬** `T = X(I − X)⁻¹`을 얻습니다 — 직접 영향과 **모든 경로를 통한 간접 영향**을 모두
더한 것입니다. 여기서 `R`(행 합)과 `C`(열 합)이 나옵니다: **`R+C`는 현저도**(체계 내 중요성), **`R−C`는 관계도**(양수 =
**원인**, 음수 = **결과**).

**일반적 용도.** 지속가능 공급망, 기술 수용의 장벽, 시스템 리스크 분석.

**🔒 이 프로젝트에서.** DEMATEL은 순위에 **앞서는** 질문에 답합니다: *"나는 어디에 손대야 하는가?"*. 그것은 **IITA(환각),
PSR(건강도), IDLS(린 낭비)가 원인**이며, **NPV, IRR, PI와 리스크 지표들은 결과**임을 드러냅니다. 반직관적이면서 해방적입니다:
NPV를 쫓아봐야 소용없습니다 — 그것은 **온도계**입니다. 환각에 손대면 NPV, IRR, 리스크가 *함께* 좋아집니다. 게다가 기준의
**가중치**는 임의로 정해지지 않고 **영향 구조에서 도출**됩니다: `wᵢ ∝ √((R+C)ᵢ² + (R−C)ᵢ²)`. 이 가중치가 **나머지 다섯
방법**을 먹입니다 — John (2025)이 서술한 통합 패턴입니다.

#### 📖 2. AHP-TOPSIS 2n — *계층분석법* + *이상해 근접 순위법*

**무엇인가.** **Saaty (1977)**가 AHP를 제안했습니다: 기준 간 **쌍대비교**로 가중치를 도출하되, 모순된 판단을 폭로하는
**일관성 검정**(`CR ≤ 0.10`)을 둡니다. **Hwang & Yoon (1981)**은 TOPSIS를 제안했습니다: 최선의 대안은 **이상해에 가장 가깝고**
**반이상해에서 가장 먼** 것입니다.

**어떻게 작동하는가.** 의사결정 행렬을 정규화하고, 열에 가중치를 곱하고, 이상해·반이상해까지의 유클리드 거리를 계산하면,
**근접 계수** `Ci = d⁻/(d⁺+d⁻)`가 전체를 정렬합니다.

**일반적 용도.** 전 세계 MCDM에서 가장 널리 쓰이는 조합입니다 — 공급업체 선정부터 성과 평가까지.

**🔒 이 프로젝트에서.** 우리는 TOPSIS를 **두 가지 정규화** — 벡터(유클리드)와 min-max(선형) — 로 돌립니다. 여기서 **"2n"**이
나옵니다. 각 프로젝트는 두 개의 계수를 받고 최종 순위는 그 평균입니다. 이로써 거의 아무도 보고하지 않는 척도를 얻습니다:
**두 정규화 간의 일치도**. 둘이 어떤 프로젝트의 위치를 두고 어긋난다면, 그 결과는 **임의의 기술적 선택에 취약**하다는 뜻이고,
대시보드가 이를 보여줍니다. 이 프로젝트의 Saaty 행렬은 `CR = 0.0119`로, 0.10 한계보다 한참 낮습니다.

#### 📖 3. ELECTRE I — *ÉLimination Et Choix Traduisant la REalité*

**무엇인가.** **베르나르 루아(1968)**, 파리의 컨설팅사 SEMA에서. 유럽 우월관계 학파의 원점입니다. 질문은 *"각자의 점수는
얼마인가?"*가 아니라 *"**a**는 적어도 **b**만큼 좋은가?"*입니다.

**어떻게 작동하는가.** 각 쌍 `(a, b)`에 대해 두 지수를 계산합니다. **일치도** `C(a,b)`는 `a`가 `b`보다 적어도 나쁘지 않은
기준들의 가중치를 더합니다. **불일치도** `D(a,b)`는 `b`에 대한 `a`의 **최대 열세**를 잽니다. 일치도가 높고 **동시에** 불일치도가
낮으면 `a`가 `b`를 **능가한다**고 말합니다. **아무도 능가하지 못하는** 대안들의 집합이 **커널**(*kernel*) — 방어 가능한
선택지의 메뉴입니다.

**일반적 용도.** 공공·환경 의사결정. 파국적인 기준을 다른 것으로 보상하는 일이 용납되지 않는 곳입니다.

**🔒 이 프로젝트에서.** ELECTRE는 **편의를 위해 거짓말하기를 거부하는** 방법입니다. NPV가 천문학적이지만 환각이 참담한
프로젝트는 자기 자리를 **사지 못합니다**: **불일치도**가 막습니다. 프레임워크는 **커널** — 다른 어떤 프로젝트도 능가하지 못하는
것들 — 을 보고하고, 점수로는 **순 우월 정도**(자신이 능가하는 수에서 자신을 능가하는 수를 뺀 값)를 씁니다. 또한 여섯 중 유일하게
이렇게 말할 자격이 있습니다: *"이 두 프로젝트는 그저 **비교 불가능**하다"*.

#### 📖 4. PROMETHEE II — *선호 순위 조직화 방법*

**무엇인가.** **장-피에르 브랑(Jean-Pierre Brans, 1982)**, **베르나르 마레샬과 필리프 뱅크(1985)**와 함께 정교화했습니다.
역시 우월관계이지만 우아한 전환이 있습니다: 이진 임계값 대신, `a`가 `b`보다 **얼마나** 선호되는지를 잽니다.

**어떻게 작동하는가.** 각 기준에서 차이 `d = g(a) − g(b)`가 **선호 함수**를 거쳐 0과 1 사이의 정도로 변환됩니다. 브랑은
**여섯 개의 일반화 함수**(통상형, 준기준형, 선호 임계형, 수준형, 무차별 포함 선형, 가우시안)를 제안했고, 이들은
**무차별 임계값 `q`**(그 아래에서는 차이가 무의미)와 **선호 임계값 `p`**(그 위에서는 선호가 완전)로 매개화됩니다. 가중된 정도를
합하면 `φ⁺`는 `a`가 남들을 능가하는 정도, `φ⁻`는 능가당하는 정도이고, **순 흐름** `φ = φ⁺ − φ⁻`가 **완전 준순서**를
낳습니다(PROMETHEE II).

**일반적 용도.** 에너지, 물류, 보건 — 선호의 **강도**를 등급화하는 것이 중요한 모든 곳.

**🔒 이 프로젝트에서.** 우리는 **무차별 포함 선형** 함수를 쓰며, `q`와 `p`는 각 기준에서 관측된 편차의 10%·90% 분위수로
추정합니다. PROMETHEE는 *"승자가 얼마나 더 나은가?"*에 답하지, 단지 *"더 나은가?"*에 답하지 않습니다. 그리고 바로 이것이 이
포트폴리오에서 가장 흥미로운 발견을 낳았습니다: 견고성 분석에서 **PROMETHEE II는 합의 선두를 오직 25.4%의 선호 우주에서만
선택**합니다 — 나머지 네 학파는 동의하는데도. 합의가 **학파 간의 불일치를 가리고 있었던** 것입니다.

#### 📖 5. MAUT — *다속성 효용 이론*

**무엇인가.** **랠프 키니 & 하워드 라이파(1976)**, 폰 노이만과 모르겐슈테른의 직계 상속자들. 공리적 형태의 미국 학파입니다:
당신의 선호가 어떤 합리성 공리들을 따른다면, 그것을 표현하는 **효용 함수**가 존재하며, 결정한다는 것은 **기대효용을
극대화**하는 일입니다.

**어떻게 작동하는가.** 각 기준은 성과를 만족으로 사상하는 **효용 함수** `uⱼ`를 받습니다. 전체 효용은 가법적입니다:
`U(a) = Σ wⱼ · uⱼ(a)` — 선호에서의 **가법적 독립성** 아래 유효합니다. 결정적인 것은 함수의 **모양**입니다: **오목한** `u`는
**위험 회피**를 나타냅니다(두 번째 백만은 첫 번째보다 덜 가치 있습니다); 선형은 중립, 볼록은 위험 선호입니다.

**일반적 용도.** 의료 의사결정, 에너지 정책, 협상 — 위험에 대한 태도를 **명시하고 방어**해야 하는 모든 맥락.

**🔒 이 프로젝트에서.** 우리는 **지수** 효용 `u(z) = (1 − e^(−r·z)) / (1 − e^(−r))`을 회피 계수 `r = 2`로 씁니다. 이는
**선언된 윤리적 선택**입니다: 이 프레임워크는 **보수적**입니다. 같은 평균의 확실한 이익보다 불확실한 이익의 가치가 낮습니다 —
신중한 CFO가 평가하는 방식 그대로입니다. TOPSIS가 모든 이익을 대체 가능한 것으로 다루는 반면, MAUT는 **높고 불확실한 약속에
벌점을 매깁니다**.

#### 📖 6. MCDA-C — *다기준 의사결정 지원 — 구성주의*

**무엇인가.** **레오나르두 엔슬린, 질베르투 몬티벨러, 산드라 노로냐(2001)**가 형식화했으며, 뿌리는 루아와 바나 이 코스타에
있습니다. 전제는 철학적입니다: 모델은 의사결정자보다 **먼저 존재하지 않습니다**. 세 단계로 **그와 함께 지어집니다** —
**구조화**(인지 지도, 서술자), **평가**(가치 함수, 대체율), **권고**.

**어떻게 작동하는가.** 각 기준은 여러 수준을 가진 **서술자**를 받고, 그중 둘이 정박점입니다: **중립** 수준(그 아래에서는 성과가
위태롭다)과 **좋음** 수준(그 위는 탁월함). 가치 함수는 정박됩니다: 중립에서 `V = 0`, 좋음에서 `V = 100`, 그리고 구간 밖으로
자유롭게 **외삽**합니다.

**일반적 용도.** 조직 성과 평가, 공공 관리, 의사결정자가 자기 문제에 대해 **배워야** 하는 맥락들.

**🔒 이 프로젝트에서.** 의사결정자와의 구조화 세션이 없으므로, 우리는 수준을 포트폴리오의 **관측 사분위수**에 정박합니다:
`중립 = Q1`, `좋음 = Q3`. 이로써 MCDA-C만의 고유함이 보존됩니다 — 그것은 단지 **정렬**하지 않고 **분류**합니다: `V < 0`는
**위태로움**, `0 ≤ V ≤ 100`은 **경쟁력 있음**, `V > 100`은 **탁월함**. 어떤 프로젝트는 순위 1위이면서도 위태로움 구간에 있을 수
있습니다. 이 묶음의 다른 어떤 방법도 그것을 말해주지 않습니다.

#### 📖 왜 다섯 방법인가, 하나가 아니라

**각 학파가 서로 다른 방식으로 틀리기** 때문이며, 홀로 선 방법은 **암묵적 100% 확신**과 함께 승자를 내놓기 때문입니다 — 그것은
언제나 거짓입니다. AHP-TOPSIS는 과도하게 보상하고, ELECTRE는 때때로 결정을 거부하며, MAUT는 효용의 모양에, MCDA-C는 정박점에
의존합니다.

우리는 **같은 가중치**(DEMATEL의 것)로 다섯을 모두 돌리고 **보르다 합의**로 닫습니다. 그러면 그들 사이의 불일치는 성가심이기를
멈추고 **정보**가 됩니다: 넷이 동의하고 하나가 정면으로 반대할 때, 그것은 잡음이 아니라 당신의 선택이 **당신이 암묵적으로 채택한
의사결정 학파에 달려 있다**는 경고입니다.

#### 📖 마지막 질문: 그 평결은 살아남는가?

위의 모든 건축물은 **가중치** 위에 서 있고, 가중치는 **추정치**입니다. IITA 가중치의 2%p가 1위와 2위를 뒤바꾼다면, 그 "승자"는
포트폴리오의 사실이 아니라 **보정의 산물**입니다.

그래서 우리는 DEMATEL 가중치를 **디리클레**로 섭동합니다 — `w' ~ Dir(κ·w)`, 정확히 단체 위에 살며 `E[w'] = w`를 보존하므로
**편향 없이** 흔듭니다 — 그리고 **2,000번** 다시 순위를 매깁니다. 평결의 성격이 바뀝니다:

> *"Project C가 최고다"* ⟶ **"Project C는 그럴듯한 선호 우주의 99.9%에서 승리한다"**

이것은 **의사결정 그 자체에 대한 신뢰구간**입니다. 이로써 프레임워크는 **돈**의 리스크만 재는 일을 멈추고 **의사결정**의 리스크를
재기 시작합니다.


**무엇이며 무엇을 위한 것인가.** 프로젝트를 고를 때 기준들은 **충돌하고**(높은 NPV는 대개 높은 리스크를 동반) 서로
**통약 불가능**합니다(원화와 환각 비율을 어떻게 더합니까?). MCDA는 그 선택을 명시적이고, 감사 가능하며, 방어 가능하게
만드는 분야입니다. 그 창립 명제는 불편하면서도 해방적입니다: **진공 속의 "최선"은 없다.** *당신이 명시한 선호 체계
아래에서의* 최선이 있을 뿐입니다. 암묵적 의견을 명시적 모델로 바꾸는 것 — 그것이 이득의 전부입니다.

**세 학파.** 가치와 효용의 **미국 학파**(AHP, MAUT): 모든 것을 하나의 숫자로 집계합니다. 베르나르 루아(Bernard Roy)의
우월관계 **유럽 학파**(ELECTRE, PROMETHEE): 두 대안이 **비교 불가능**할 수 있음을 받아들이고 **거부권**을 허용합니다 —
한 기준의 형편없는 점수는 다른 기준의 뛰어난 점수로 매수되지 않습니다. **구성주의 학파**(MCDA-C): 모델은 발견되는 것이
아니라 **의사결정자와 함께 지어지는** 것입니다.

| 방법 | 기원 | 핵심 질문 | 오직 그것만이 주는 것 | AI 포트폴리오에서 |
|---|---|---|---|---|
| **DEMATEL** | Gabus & Fontela, Battelle (1972-73) | *"누가 누구에게 영향을 주는가?"* | **원인**과 **결과**를 가르고, 영향 구조 자체에서 **가중치**를 도출 | 환각 감소(IITA)가 **원인**임을 보여줌 — 거기를 건드리면 NPV, IRR, 리스크가 함께 좋아짐 |
| **AHP-TOPSIS 2n** | Saaty (1977) · Hwang & Yoon (1981) | *"누가 이상해에 가장 가까운가?"* | 쌍대비교 가중치 + **일관성 검정**(CR ≤ 0.10) | **두 정규화**로 순위를 매기고 둘의 일치도를 보고 |
| **ELECTRE I** | Bernard Roy (1968) | *"누가 누구를 능가하며, 누가 지배당하지 않고 살아남는가?"* | **비교 불가능성**과 **거부권**: 형편없는 기준은 매수되지 않음 | 어떤 프로젝트에도 지배당하지 않는 **커널**을 분리 |
| **PROMETHEE II** | Brans & Vincke (1985) | *"순 선호 흐름은 얼마인가?"* | 무차별·선호 임계값을 가진 **여섯 개의 선호 함수** | *얼마나* 더 나은지를 등급화, 단지 *더 나은지*가 아니라 |
| **MAUT** | Keeney & Raiffa (1976) | *"의사결정자의 효용을 무엇이 극대화하는가?"* | 오목 효용으로 **위험 회피**를 모델링 | 불확실한 이익에 벌점 — 신중한 결정권자는 같은 값을 치르지 않음 |
| **MCDA-C** | Ensslin, Montibeller & Noronha (2001) | *"'좋음' 수준은 어디이고 '중립'은 어디인가?"* | **정박된 가치함수**: 중립에서 `V=0`, 좋음에서 `V=100`, 외삽 허용 | 단순 정렬이 아니라 **위험 / 경쟁력 / 탁월**로 분류 |

**왜 다섯인가, 하나가 아니라.** 각 학파는 서로 다른 방식으로 틀립니다. 하나의 방법은 **암묵적 100% 확신**과 함께 승자를
내놓는데, 그것은 언제나 거짓입니다. 다섯을 모두 돌리고 **보르다 합의**로 닫으면, 그들 사이의 불일치가 **정보**가 됩니다:
넷이 동의하고 하나가 정면으로 반대할 때, 그것은 잡음이 아니라 경고입니다 — 당신의 선택이 *우월관계*를 *효용*보다
선호하는지에 달려 있다는. 그리고 가중치에 가한 **디리클레 섭동**이 마지막 질문에 답합니다: *"1위가 보정의 2%p 오차를
견디는가?"*


### 🧪 네 개의 톱니바퀴: Iman-Conover, 콜모고로프-스미르노프, 디리클레, 그리고 토네이도

위의 두 큰 방법은 네 개의 더 작은 부품 위에 서 있습니다 — 그리고 정직한 시뮬레이션과 예쁜 숫자를 가르는 차이가 바로 그
부품들 안에 있습니다. 알아둘 가치가 있습니다.

#### 🔗 Iman-Conover — **분포를 파괴하지 않고** 상관을 부여하기

**무엇인가.** **Ronald Iman과 William Conover (1982)**가 제안했습니다. 사소해 보이지만 그렇지 않은 문제를 풉니다:
*주변분포가 정규가 아닐 때 상관된 변수를 어떻게 추출하는가?* 순진한 길 — 촐레스키로 상관된 정규를 만들고 변환하기 — 는
**주변분포를 일그러뜨립니다**. 그리고 방금 데이터에 LogNormal을 적합시켰다면, 그것을 일그러뜨리는 일은 방금 한 작업을 그대로
버리는 것입니다.

**어떻게 작동하는가.** 이것은 값의 변환이 아니라 **순위 재정렬**입니다. **판 데르 바르덴 점수** `Φ⁻¹(i/(n+1))`로 참조를 만들고
열마다 섞습니다. `P = chol(R)`(목표)과 `Q = chol(corr(M))`(참조의 우연한 상관)을 계산하고 `S = M·(Q⁻¹P)ᵀ`을 만듭니다. 그런 다음
원표본의 각 열을 **`S`의 순위에 따라 재정렬**합니다. 이미 뽑힌 값들의 *순서*만 바뀌므로 **주변분포는 정확히 보존**됩니다 —
비트 단위로.

**미세하고 정직한 한 가지.** `R`은 *정규 참조*의 상관이지, 결과의 피어슨 상관이 아닙니다. 유도된 순위 상관은 정규 코퓰러를
따릅니다: `ρ_S = (6/π)·arcsin(R/2)`. `R = 0.80`이면 **0.7859**이며 — 이는 우리가 시험에서 측정한 값(0.786)과 정확히 일치합니다.
이는 방법의 오류가 아니라, 그 수학입니다.

**일반적 용도.** 금융 리스크(상관 자산), 구조 신뢰성, 라틴 하이퍼큐브 표집.

**🔒 이 프로젝트에서.** 당신의 토큰에 적합된 분포를 **희생하지 않고** 현금흐름을 상관시키게 해 주는 것이 바로 이것입니다.
사용 전에 행렬은 검증됩니다: 대칭, 대각 1, 그리고 **양정치**(촐레스키로). 일관되지 않은 상관행렬은 최소 고윳값을 보고하며
거부됩니다 — 조용히 무의미한 숫자를 만들어내는 대신에.

#### 📏 콜모고로프-스미르노프 — 당신이 **가정한 것**과 데이터가 **말하는 것** 사이의 거리

**무엇인가.** **비모수** 적합도 검정입니다. 통계량은 단순하고 아름답습니다: `D = sup |Fₙ(x) − F(x)|`, 당신 데이터의
**경험분포함수**와 당신이 제안한 **이론분포** 사이의 최대 수직 간격. 귀무가설 아래에서 `D`의 분포는 **`F`가 무엇이든 상관없이
동일**합니다 — 그래서 *distribution-free*입니다.

**방법론적 정직성의 단서.** 고전적 KS의 p-값은 `F`의 모수가 데이터를 보기 **전에** 고정되었다고 가정합니다. 모수가 **같은
데이터에서 추정**될 때(여기처럼 최대우도로), 이 검정은 **낙관적**이 됩니다: 너무 쉽게 받아들입니다. 엄밀함은 **릴리포스**
보정이나 **모수적 부트스트랩**을 요구할 것입니다. 그래서 우리는 KS를 증명이 아니라 **진단**으로 다루며, 나쁜 적합을
**기각**하는 데에만 쓰고, 어떤 적합을 "옳다"고 선언하는 데에는 결코 쓰지 않습니다.

**일반적 용도.** 적합도; 두 표본 비교(이표본 KS); 프로덕션 머신러닝 시스템의 데이터 *드리프트* 탐지.

**🔒 이 프로젝트에서.** AIC로 승리한 분포가 실제로 당신의 토큰 시계열을 얼마나 잘 설명하는지 잽니다. p-값이 0.05 아래로
떨어지면 화면은 **`적합도 취약`을 붉게** 찍습니다 — 데모 포트폴리오에서 실제로 한 프로젝트에 이런 일이 일어나며,
프레임워크는 그것을 숨기지 않고 **보여줍니다**. 예쁜 숫자보다 정직한 숫자가 낫습니다.

#### 🎲 디리클레 섭동 — **의사결정의 신뢰구간**

**무엇인가.** **디리클레** 분포는 **단체(simplex)** 위의 자연스러운 분포입니다: 합이 1인 양수 벡터 — 바로 가중치 벡터가 그것입니다.
다항분포의 켤레이며 베타 분포의 일반화입니다.

**왜 가우시안 잡음이 아니라 이것인가.** 가중치에 정규 잡음을 더하면 음수가 생기고 합이 1이라는 제약이 깨집니다. 디리클레는 유효
공간 *안에서* 삽니다. 그리고 `w' ~ Dir(κ·w)`로 매개화하면 이 일에 완벽한 두 성질을 가집니다: `E[w'] = w`(**편향 없이** 섭동)와
`Var(w'ᵢ) = wᵢ(1−wᵢ)/(κ+1)`(분산을 손잡이 하나로 조절). `κ → ∞`이면 원래 가중치로 붕괴합니다.

**일반적 용도.** 비율에 대한 베이지안 *사전분포*; 토픽 모델링의 잠재 디리클레 할당(**LDA**); 루빈(1981)의 **베이지안 부트스트랩**;
그리고 다기준 의사결정에서의 가중치 민감도 분석.

**🔒 이 프로젝트에서.** `κ = 200`일 때 13%의 가중치는 약 **±2.37%p** 흔들립니다 — 전문가 판단의 그럴듯한 오차 범위입니다.
**2,000번** 다시 순위를 매겨 각 프로젝트의 `P(승리)`를 얻습니다. 포트폴리오에서 가장 불편한 발견을 드러낸 것이 바로 이 톱니바퀴였습니다:
합의는 견고하지만(99.9%), **PROMETHEE II는 우주의 25.4%에서만 선두를 선택**합니다. 디리클레가 없었다면 그 불일치는 보이지 않았을 것입니다.

#### 🌪️ 민감도 토네이도 — 어떤 변수가 **정말로** 결과를 움직이는가

**무엇인가.** 절대 효과 크기로 정렬된 수평 막대그래프로, *불확실한 입력들 가운데 무엇이 출력을 움직이는가?*에 답합니다.
이름은 그 모양에서 왔습니다 — 위는 넓고 아래는 좁습니다.

**같아 보이지만 같지 않은 두 척도.**
- 다중회귀의 **베타**는 이렇게 답합니다: *"이 입력이 1단위 오르면 출력은 얼마나 오르는가?"* 그것은 **단위** 효과이며, 그 입력이
  실제로 얼마나 변동하는지에는 무관심합니다.
- **피어슨 상관**은 이렇게 답합니다: *"출력의 불확실성 가운데 얼마가 이 입력에 의해 좌우되는가?"* 이미 **불확실성의 규모**를
  포함하고 있습니다(대략 `β·σᵢ/σ_y`).

어떤 변수는 베타가 거대하면서 상관은 0일 수 있습니다: 결과를 크게 *움직일* 테지만, 실제로는 **거의 변동하지 않는** 것입니다.
둘 중 하나만 보고하는 것은 절반의 진실입니다.

**일반적 용도.** 프로젝트 리스크, 금융 모형, 신뢰성 공학, 시뮬레이터 보정.

**🔒 이 프로젝트에서.** 여기서 토네이도는 드문 일을 했습니다: **모형 자체의 한계를 폭로**했습니다. NPV에 대해 돌리자 베타들이
**정확히 `1/(1+i)ᵗ`** — 할인계수 — 로 나왔습니다. NPV가 현금흐름에 *선형*이기 때문입니다. 이 경우 회귀 토네이도는 **퇴화**합니다:
할인율 외에 아무것도 말하지 않습니다. 신호를 지니는 것은 **상관**입니다. 그리고 토큰 비용이 변수로 들어오자, 그 베타는
`−1/(1+i)ᵗ`(비용은 음의 부호로 들어옵니다)이고 상관은 0에 가까웠습니다. 함께 읽으면 진술은 정확하고 정직합니다:
*"토큰에 1단위를 더 쓰면 NPV는 0.91 줄어든다 — 그러나 이 프로젝트에서 NPV의 불확실성은 토큰에서 오지 않는다."*
어느 한 척도만으로는 이 말을 할 수 없습니다.

## 🙋 반론 (지금 당신이 스스로에게 묻는 질문들)

- **"시간이 없어요."** → `./run_all.sh --mock`로 5분이면 대시보드가 화면에서 돕니다. 측정은 당신이 이미 재작업과 환각에
  잃고 있는 시간을 **돌려줍니다.**
- **"너무 복잡해요."** → 한 줄이면 됩니다. 프레임워크가 ETL, 계산, 순위, 이미지를 처리하고, **당신은 평결만 읽습니다.**
- **"제 AI 운영은 작아요."** → 바로 그래서 1달러가 더 무겁습니다. 오늘은 작아도 내일은 포트폴리오 — **낭비를 확장하기 전에
  측정하세요.**
- **"저는 Langfuse를 안 써요."** → 데모는 **Langfuse 없이 100% 작동**합니다. 실제 데이터를 원하면 **당신의** 계정을
  연결하세요(제 것이 아니라).
- **"그냥 또 하나의 예쁜 대시보드 아닌가요."** → 아닙니다. **균형성과표 + 투자 분석(NPV/IRR/MIRR/EAA) + 다기준
  의사결정(AHP-TOPSIS)** — 장식이 아니라 이사회급 도구입니다.
- **"제 데이터 프라이버시는요?"** → 데모는 **100% 익명**(Project A…J)이며, 실제 데이터/이름과 키는 **패키지 밖**에 있습니다.
  당신은 **로컬**에서, **당신의** 계정으로 실행합니다.
- **"얼마인가요?"** → **무료.** 오픈 소스, 당신의 컴퓨터에서. 유일한 대가는 계속 **측정하지 않는 것** — 그리고 그건 이미 내고
  있습니다.

## 🧰 처음부터 시작하는 로컬 설치 단계

> 모든 것이 **당신의 컴퓨터에서** 돌아갑니다. 작성자의 키는 패키지에 포함되지 않으며, 어떤 데이터도 밖으로 나가지 않습니다.

### 0단계 — 사전 요구사항

| 요구사항 | 버전 | 필수? | 용도 |
|---|---|---|---|
| **Python** | 3.10+ | ✅ | 파이프라인, KPI, 몬테카를로, MCDM |
| **Node.js + npm** | 18+ | ✅ | 대시보드(Evidence) |
| **git** | 아무거나 | ✅ | 저장소 복제 |
| **Rust + maturin** | 안정판 | ⬜ 선택 | 로그 분류 가속 |
| **tectonic** | 아무거나 | ⬜ 선택 | PDF 피치 덱 생성 |

*Windows에서는 **WSL** 또는 **Git Bash**를 사용하세요 — 파이프라인은 `bash` 스크립트입니다.*

### 1단계 — 저장소 복제
```bash
git clone https://github.com/bpenedo/Gestao-de-Projetos-PM-IA-BSC-DashBoard.git
cd Gestao-de-Projetos-PM-IA-BSC-DashBoard
```

### 2단계 — 격리된 Python 환경
```bash
cd foundations/pipeline
python3 -m venv .venv
source .venv/bin/activate        # Windows (PowerShell): .venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 3단계 — 대시보드 의존성
```bash
cd ../evidence
npm install
```

### 4단계 — 데모 실행(익명, 자격증명 불필요)
```bash
cd ../pipeline
./run_all.sh --mock
```

순서대로: 익명 데모 데이터 → KPI → NPV/IRR/MIRR/EAA/PI → **토큰에 분포 적합** → **몬테카를로(10,000회)** →
AHP-TOPSIS 2n → **DEMATEL · ELECTRE · PROMETHEE · MAUT · MCDA-C** → **순위 견고성(디리클레)** → 그래프 →
관리 도시에 → 5D 지도 → 피치 덱 → 대시보드 빌드.

### 5단계 — 대시보드 열기
```bash
cd ../evidence
npm run dev          # http://localhost:3000
npm run preview      # (alternativa) serve o estático já compilado em build/
```

### 6단계 — 당신의 데이터로 전환

**6.1 — 자격증명과 파라미터**(모두 선택; `.env`가 없으면 기본값 사용):
```bash
cd ../pipeline
cp .env.example .env      # edite: LANGFUSE_PUBLIC_KEY, LANGFUSE_SECRET_KEY, SELIC_ANUAL, USD_BRL...
```

**6.2 — 당신의 현금흐름**(NPV, IRR, 몬테카를로를 먹이는 것):
```bash
cp fluxo_caixa_template.csv fluxo_caixa.csv
```
형식: `periodo 0`은 투자(음의 흐름), `taxa`는 기간당 할인율(`0.10` = 10%).
```csv
project_name,periodo,fluxo,taxa
Project A,0,-12000,0.10
Project A,1,3000,0.10
Project A,2,4000,0.10
```

**6.3 — 실제 데이터로 실행:**
```bash
./run_all.sh          # sem --mock: sincroniza do Langfuse e usa fluxo_caixa.csv
```

### 7단계(선택) — 가속과 PDF
```bash
cd analise_rs && maturin develop --release && cd ..   # Rust (PyO3): classificação mais rápida
```
피치 덱을 원하면 **tectonic**을 설치하세요(예: `cargo install tectonic` 또는 배포판 패키지 관리자).

### 8단계(선택) — 갱신 예약
```bash
crontab -e
*/15 * * * * /CAMINHO/ABSOLUTO/foundations/pipeline/run_all.sh >> /tmp/bsc.log 2>&1
```

### 🩺 자주 겪는 문제

| 증상 | 유력한 원인 | 해결 |
|---|---|---|
| `no such table: ...` | DB 미초기화 | `python3 db.py` |
| 대시보드 빌드 실패 | 오래된 산출물 | `rm -rf ../evidence/build && npm run build` |
