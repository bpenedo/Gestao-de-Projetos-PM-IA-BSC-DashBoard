### 💸 אתם משלמים על AI כל חודש. אבל האם ה-AI משלם **לכם** בחזרה?

בכל פעם שהכרטיס מחויב על ידי **ChatGPT, Claude, Copilot, Gemini, Perplexity, DeepSeek, Kimi, Qwen…**, נותרת שאלה
של **מיליונים** ללא מענה: **היכן התשואה?** כמה שעות-אדם נחסכו באמת? כמה מכספכם **התאדה** להזיות, עבודה חוזרת
והמתנה? איזה פרויקט AI **ראוי להתרחב היום** — ואיזה **מדמם מזומנים** בעודכם מוחאים כף ל"חדשנות"?

אין לכם עלות AI. יש לכם **נזילה שקטה** — ועיניכם מכוסות. כי *"מה שלא נמדד לא ניתן לניהול ולא לשיפור"* — והשוק מודד
זאת עבורכם, ושולח לכם את החשבון.

**המסגרת הזו מדליקה את האור.** היא הופכת את **ההוצאה הבלתי-נראית** של מנויי ה-AI שלכם ל**תשואה מדידה, ברת-השוואה
וברת-ביקורת** — בקפדנות של **Balanced Scorecard** (Kaplan & Norton), **ניתוח השקעות ברמת וול-סטריט** ו**החלטה
רב-קריטריונית**. זה ההבדל בין *לקוות* לבין *לדעת*. בין לשלם על AI לבין **להרוויח** ממנו.

## 🌅 מדוע זה משנה את המשחק

**יש שני סוגי אנשים בעידן ה-AI.** הראשונים נרשמים להכול, מוציאים בגדול ו**מתפללים** שיעבוד — ומעבים את הסטטיסטיקה
האכזרית של פרויקטים שמתים בפיילוט. השניים עושים מה שוול-סטריט עושה עם כל נכס רציני: הם **מודדים, משווים, מתעדפים
ומקצים מחדש** — והופכים כל דולר מנוי ל**תשואה מצטברת**. ההבדל היחיד ביניהם **אינו כישרון ואינו תקציב. הוא
אינסטרומנטציה.**

ה-AI הגנרטיבי יצר סוג חדש של הוצאה חוזרת — **מנויים וטוקנים** — ואיתו את הבזבוז היקר ביותר של העשור: **הבלתי-נראה.**
מה שאינכם רואים, אינכם מתקנים. מה שאינכם מודדים, אינכם מרחיבים. ומה שאינכם מוכיחים, הדירקטוריון אינו מאשר.

**הפרויקט הזה מעביר אתכם מהשבט הראשון לשני.** הוא ממכשר כל פרויקט AI כ**נכס פיננסי** ומודד אותו תחת **Balanced
Scorecard**, **ניתוח השקעות (NPV, IRR, MIRR, EAA, PI, החזר)** ו**Lean Six Sigma** — ואף **בוחר את הפרויקט הטוב
ביותר בתיק שלכם** באמצעות מודל רב-קריטריוני (**AHP-TOPSIS 2n**). החשבון החודשי האטום הופך ל**תזת השקעה ברת-ביקורת**:
אתם מגלים, במספרים, היכן להרחיב, היכן לחתוך, היכן המנוי מחזיר את עצמו תוך **שבועות** — והיכן הוא מדמם בלי כיסוי.

אנו **חלוצים** של טריטוריה חדשה — **הגבול שבין הבינה המלאכותית לבינה הפיננסית עתירת-הערך**. כמו חוקרים הממפים ארצות שטרם מופו,
המסגרת הזו היא ה**מצפן** (🧭) ההופך את ערפל המנויים ל**נתיבי תשואה ברורים**: כל טוקן — מייל; כל פרויקט — משלחת אל
הרווח. היכן שהיתה עלות עיוורת, נולדת **הזדמנות מדידה**; היכן שהיתה גיליון מת, פורחת **תזת השקעה חיה**.

### 🚀 עבור מיקרו-SaaS, SaaS, סטארט-אפים וסולופרנרים

הנה מה שאיש לא סיפר לכם כששילבתם AI במוצר: **בדיוק העברתם את ה-AI מתקציב השיווק אל ה-COGS שלכם.**
ו-COGS שגדל עם השימוש אינו הוצאה — הוא **משכנתה על הרווח הגולמי שלכם**. כל משתמש חדש עולה עכשיו טוקנים.
כל ניסיון חוזר בגלל הזיה שורף רווחיות פעמיים. והחשבון מגיע רק בסוף החודש, כשכבר אי-אפשר לבטל.

| אתם… | הכאב שאיש אינו מודד | מה המסגרת הזו מחזירה |
|---|---|---|
| **סולופרנר** | אתם *הצוות*; שעת העבודה שלכם היא הנכס היקר ביותר שקיים | ה**טורנדו** מצביע על המשתנה שמזיז את התוצאה — כלומר **היכן להשקיע את השעה הבאה** |
| **מיקרו-SaaS** | עלות הטוקנים גדלה עם השימוש ואוכלת את הרווחיות בשקט | התפלגות **מותאמת לטוקנים האמיתיים שלכם** + **CVaR**: לחודש הרע יש מחיר *לפני* שהוא מגיע |
| **SaaS בקנה מידה** | כל פיצ'ר AI הוא פרויקט הנאבק על אותו מפת דרכים | **חמש שיטות** בוחרות מי יוצא — וה**חוסן** אומר אם המקום הראשון שורד טעות של 2 נקודות במשקל |
| **סטארט-אפ שמגייס** | המשקיע אינו קונה *"אנחנו משתמשים ב-AI"* | הוא קונה **NPV, IRR, החזר ו-`P(NPV<0)`** — וה**pitch deck יוצא מוכן**, ב-LaTeX |

**מה באמת משתנה.** הרווח הגולמי שלכם חדל להיות הערכה והופך ל**התפלגות עם זנב מתומחר**. ה-runway חדל להיות חילוק
פשוט ומקבל **VaR**: *"ב-19 מתוך 20 תרחישים, המזומן שלי מחזיק לפחות N חודשים."* וכשמגיע הבורד-דק או בדיקת הנאותות,
אתם לא פותחים גיליון שאיש אינו יכול לשחזר — אתם פותחים מספר עם **זרע קבוע**, שכל שותף, משקיע או מבקר מריץ שוב ומקבל
**בדיוק אותו דבר**.

> **מיצוב מחדש אכזרי:** הסולופרנר מתחיל להחליט כמו CFO. וה-CFO מתחיל להחליט במהירות של סולופרנר.

> **ההבטחה:** להפוך את מי ש*משלם על AI* למי ש*מרוויח מ-AI* — ואת מי ש*משתמש ב-AI* למי ש**שולט בו, מודד אותו ומכפיל
> אותו כחלוץ**. במספרים, לא באמונה.

## 📈 הראיות (Gartner · IDC · PwC · McKinsey · MIT)

אל תאמינו לי. **תאמינו למכונים החוקרים זאת עשרות שנים** — ופסק דינם פה-אחד: **ה-AI יוצר ערך עצום, אך מספק אותו רק
למי שמודד ומנהל.** מי ש"משתמש ב-AI בלי לשלוט בו" הופך לסטטיסטיקת נטישה; מי שממכשר את התשואה **זוכה בפרס**.

- 🧭 **Gartner** — חזה כי **≥ 30% מפרויקטי ה-AI הגנרטיבי ינוטשו לאחר הוכחת ההיתכנות עד סוף 2025**, כשהסיבה
  המרכזית היא **ערך עסקי לא-ברור** (לצד נתונים גרועים, עלויות מטפסות ובקרות רופפות). *→ בלי מדידה, הפרויקט מת
  בפיילוט.*
- 🔬 **MIT** (דוח *"The GenAI Divide / State of AI in Business 2025"*, יוזמת NANDA) — דווח בהרחבה כי **הרוב המכריע
  של פיילוטי GenAI ארגוניים אינו מייצר השפעה מדידה על הרווח וההפסד**; המיעוט המספק ערך משלב AI עם **תהליך ומדידה**.
  *→ ההבדל הוא מדידה, לא אימוץ.*
- 💵 **IDC** (מחקר *"The Business Opportunity of AI"*, בחסות Microsoft) — ארגונים ש**מודדים ומייעלים** דיווחו על
  תשואה בסדר גודל של **כמה דולרים על כל 1 US$** שהושקע ב-AI, עם פיזור רב בין מובילים למפגרים. *→ ה-ROI קיים —
  ומעדיף את מי שממכשר.*
- 🌍 **PwC** (*"Sizing the Prize"*) — מעריך שה-AI עשוי להוסיף עד **~15.7 טריליון US$** לכלכלה העולמית עד 2030; אך
  הפרס מגיע למי ש**לוכד** את הערך, לא למי שרק צורך אותו. *→ העוגה ענקית; הפרוסה שייכת למי שמודד.*
- 🏆 **McKinsey** (*"The State of AI"*) ו-**BCG × MIT Sloan** — קבוצת מיעוט של **"AI high performers"** לוכדת
  תשואה בלתי-פרופורציונלית; נקודת המפנה מגיעה כשה-AI מחובר ל**מדדים, ממשל והשקעה חוזרת** במקום שבו התשואה מוכחת.
  *→ המנצחים מודדים, מתעדפים ומקצים מחדש.*

> **זהו בדיוק הפער שהמסגרת הזו חוצה:** היא מוציאה אתכם מהצד ש*נוטש בפיילוט* אל הצד עם **תוצאות ממשיות ומוכחות** — עם
> BSC, ניתוח השקעות והחלטה רב-קריטריונית.

> ⚠️ **הערת יושר (קראו):** המספרים לעיל משקפים כותרות אמיתיות של מכונים אלו, אך **דוחות ואחוזים מתעדכנים** — ודאו
> את הערכים המדויקים ואת השנה ב**מקורות הראשוניים** (Gartner Newsroom; IDC/Microsoft *Business Opportunity of AI*;
> PwC *Sizing the Prize*; McKinsey *State of AI*; MIT *State of AI in Business*) לפני ציטוט בחומר רשמי. כאן הם
> משמשים **ביסוס כיווני**, לא ערובה מספרית.

## 💥 מחיר חוסר-המעש (עשו את החשבון שאיש אינו עושה)

מנוי **PRO של AI** עולה בין **20 ל-200 US$ לחודש, למושב**. הכפילו במספר האנשים בצוות. הכפילו ב-12 חודשים. עכשיו
החילו את מה שהמכונים כבר **הוכיחו**: **Gartner** צופה **≥ 30% נטישה** ו-**MIT** מראה שה**רוב של הפיילוטים אינו
מחזיר**. חלק עצום מהסכום הזה אינו השקעה — הוא **דימום טהור**.

> **דוגמה ישירה (החליפו במספרים שלכם):** 10 מושבים × 30 US$/חודש × 12 = **3,600 US$/שנה**. אם ~30% הופכים לבזבוז
> בלתי-נראה, אלו **~1,080 US$/שנה שמתאדים** — מצוות קטן אחד, בשנה אחת. עם המספר האמיתי שלכם, ההלם גדול יותר.

והנה החלק שכואב: **העלות הזו מצטברת ואינה ממתינה.** כל חודש בלי מדידה הוא חודש של נזילה ש**אינה חוזרת** — בעוד
המתחרה שכבר מכשר **מקצה מחדש הון לְמה שמניב**. יתרון החלוץ נבנה מוקדם: **מי שמודד ראשון, מרחיב ראשון.**

הרגע הזול ביותר להתחיל היה אתמול. השני-הטוב-ביותר הוא **עכשיו** — והוא עולה **0 US$** ו-**5 דקות**. השאלה אינה *"האם
אני יכול להרשות לעצמי למדוד?"*. היא ***"עוד כמה זמן אני יכול להרשות לעצמי לא למדוד?"***

## 🎲 מונטה קרלו — הסיכון שהממוצע מסתיר

NPV חיובי **בממוצע** אינו מגן על איש. הממוצע הוא השקר הנוח ביותר במימון: הוא מתאר תרחיש שאולי לעולם לא יקרה. את גורלכם
קובעת ה**זנב** — היום הרע.

המסגרת הזו מדמה **10,000 עתידים** לכל פרויקט: כל תזרים מזומנים
הופך ל**משתנה מקרי**, והתיק כולו מחושב מחדש איטרציה אחר איטרציה. בסוף אין לכם מספר — יש לכם **את כל התפלגות הכסף שלכם**:

- **`P(NPV < 0)`** — ההסתברות האמיתית להפסד. המספר שאיש אינו מראה לכם.
- **VaR 5%** — התרחיש הגרוע הסביר: *"ב-19 מתוך 20 עתידים, אני מרוויח לפחות את זה."*
- **CVaR 5%** — כשהאסון מתרחש, כמה הוא עולה בממוצע.
- **טורנדו רגישות** — רגרסיה מרובה ומתאם פירסון: איזה משתנה באמת מזיז את ה-NPV שלכם.
- **20 התפלגויות** קלט, **מטריצת מתאם** מאומתת (Iman-Conover, המשמרת במדויק את ההתפלגויות השוליות) ו**אחוזונים
  מ-1% עד 99%**, עם היסטוגרמה של 100 מחלקות.

זרע קבוע: הרצה חוזרת נותנת **בדיוק** את אותה תוצאה. בר-ביקורת — לא "קסם".

> **המפנה:** אתם מפסיקים לבחור את הפרויקט עם ה-NPV הגבוה ביותר ומתחילים לבחור **את זה ששורד את התרחיש הרע**.
> זו ניהול סיכונים — זה מה שמפריד בין המשקיע למהמר.

## 🔬 האות נמצא במעלה הזרם — ושם שוכן המינוף

גיליתי זאת בעודי מודד את המסגרת עצמה: טורנדו הרגישות של ה-NPV החזיר **בדיוק**
`1.0 · 0.9091 · 0.8264 · 0.7513…` — מקדמי ההיוון `1/(1+i)ᵗ`. מכיוון שה-NPV **לינארי** בתזרימי המזומנים, סימולציה של
התזרימים בלבד אינה מלמדת דבר מעבר לשיעור. **האות האקראי האמיתי נמצא במעלה הזרם: בטוקנים.**

### 1️⃣ הפסיקו לקבוע את ההתפלגות. התאימו אותה לנתונים שלכם.

אחת-עשרה התפלגויות מועמדות מותאמות ב**נראות מרבית** לסדרת צריכת הטוקנים האמיתית שלכם (`logs_langfuse`). מנצחת זו
בעלת ה-**AIC הנמוך ביותר** — ה-AIC מעניש כל פרמטר מיותר ומונע התאמת-יתר — ומבחן **קולמוגורוב-סמירנוב** מודד את טיב
ההתאמה. זהו *התאמת התפלגויות לנתונים* הקלאסית, והיא שחושפת את **הזנב הכבד** של הצריכה: יש פרומפטים שעולים פי 10
מהטיפוסי, והזנב הזה הוא שמפוצץ את התקציב — בלתי-נראה למי שעובד עם ממוצעים.

**וכשההתאמה גרועה, המסגרת צועקת.** אם ערך ה-p של KS יורד מתחת ל-0.05, המסך מזהיר `התאמה חלשה` באדום, במקום להעמיד
פנים של דיוק. מספר כן עדיף על מספר יפה.

### 2️⃣ האם הדירוג שלכם שורד טעות של 2 נקודות במשקל?

כל שיטה רב-קריטריונית מחזירה מנצח בביטחון **מובלע של 100%**. אך משקלי הקריטריונים הם אומדנים, לא אמיתות שהתגלו. אם שתי
נקודות אחוז במשקל ה-IITA מחליפות בין המקום הראשון לשני, ה"מנצח" הוא ארטיפקט של הכיול.

לכן אנו מטרידים את משקלי ה-DEMATEL בהתפלגות **דיריכלה** — `w' ~ Dir(κ·w)`, שחיה בדיוק על הסימפלקס ומשמרת
`E[w'] = w`, כלומר מטרידה **בלי להטות** — ומדרגים מחדש **2,000 פעמים**. פסק הדין משנה את טבעו:

> *"Project C הוא הטוב ביותר"* ⟶ **"Project C מנצח ב-99.9% מיקומי ההעדפה הסבירים"**

זהו **רווח בר-סמך על ההחלטה עצמה**. והוא חושף את מה שהקונצנזוס הסתיר: במסך שלהלן, **PROMETHEE II בוחר במוביל רק
ב-25.4% מהיקומים**. ארבע אסכולות מסכימות; אחת חולקת חזיתית. זה אינו רעש — זו האזהרה שהבחירה תלויה בשאלה אם אתם מעדיפים

### 💼 מתוכנית ב-20 US$ ועד חוזה ב-200 אלף US$

**אם אתם פרילנסרים או עסק קטן:** ההתפלגות המותאמת אומרת לכם **כמה יעלה חודש הטוקנים הרע** — לפני שהוא מגיע; והחוסן
אומר אם באמת כדאי להעביר את המאמץ לפרויקט האחר, או ששניהם שווים בתוך שולי הטעות. אתם מפסיקים לייעל בחושך עם תזרים דחוק.

**אם אתם חברה גדולה:** `P(ניצחון)` הוא החלק החסר בוועדת ההשקעות. הוא הופך את *"צוות A תומך בפרויקט X"* ל-**"פרויקט X
מנצח תחת 99.9% מכיולי המשקל הברי-הגנה, והאסכולה היחידה שחולקת היא ההעדפה העודפת, בקריטריון Y"**. ויכוח פוליטי הופך
ל**ויכוח טכני** — וה-CFO מקבל מספר ששורד ביקורת.

> **המפנה האחרון:** המסגרת מפסיקה למדוד את הסיכון של ה**כסף** ומתחילה למדוד את הסיכון של ה**החלטה עצמה**.
> מעט מאוד מקומות בעולם עושים זאת.

## 🙋 התנגדויות (השאלות שאתם שואלים את עצמכם עכשיו)

- **"אין לי זמן."** → חמש דקות עם `./run_all.sh --mock` והדשבורד רץ על המסך שלכם. מדידה **מחזירה** לכם את השעות
  שאתם כבר מאבדים על עבודה חוזרת והזיות.
- **"זה מסובך מדי."** → שורה אחת. המסגרת עושה את ה-ETL, החישובים, הדירוג והתמונות; **אתם רק קוראים את פסק הדין.**
- **"מבצע ה-AI שלי קטן."** → בדיוק לכן כל דולר שוקל יותר. קטן היום, תיק מחר — **מדדו לפני שתרחיבו את הבזבוז.**
- **"אני לא משתמש ב-Langfuse."** → ההדגמה רצה **100% בלי Langfuse**. כשתרצו נתונים אמיתיים, תחברו את **החשבון
  שלכם** (לעולם לא שלי).
- **"זה סתם עוד דשבורד יפה."** → לא. זה **Balanced Scorecard + ניתוח השקעות (NPV/IRR/MIRR/EAA) + החלטה
  רב-קריטריונית (AHP-TOPSIS)** — כלֵי דירקטוריון, לא קישוט.
- **"ומה עם פרטיות הנתונים שלי?"** → ההדגמה **אנונימית ב-100%** (Project A…J); נתונים/שמות אמיתיים ומפתחות נשארים
  **מחוץ לחבילה**. אתם מריצים **מקומית**, עם **החשבון שלכם**.
- **"כמה זה עולה?"** → **כלום.** קוד פתוח, על המחשב שלכם. המחיר היחיד הוא להמשיך **לא למדוד** — ואותו אתם כבר משלמים.

## 🧰 התקנה שלב אחר שלב (מקומית, מאפס)

> הכול רץ **על המחשב שלכם**. אף מפתח של המחבר אינו מלווה את החבילה, ושום נתון אינו יוצא מהמחשב שלכם.

### שלב 0 — דרישות מוקדמות

| דרישה | גרסה | חובה? | לשם מה |
|---|---|---|---|
| **Python** | 3.10+ | ✅ | פייפליין, KPI, מונטה קרלו, MCDM |
| **Node.js + npm** | 18+ | ✅ | דשבורד (Evidence) |
| **git** | כל גרסה | ✅ | שכפול המאגר |
| **Rust + maturin** | יציב | ⬜ אופציונלי | מאיץ את סיווג הלוגים |
| **tectonic** | כל גרסה | ⬜ אופציונלי | מפיק את ה-pitch decks ב-PDF |

*ב-Windows השתמשו ב-**WSL** או ב-**Git Bash** — הפייפליין הוא סקריפט `bash`.*

### שלב 1 — שכפול המאגר
```bash
git clone https://github.com/bpenedo/Gestao-de-Projetos-PM-IA-BSC-DashBoard.git
cd Gestao-de-Projetos-PM-IA-BSC-DashBoard
```

### שלב 2 — סביבת Python מבודדת
```bash
cd foundations/pipeline
python3 -m venv .venv
source .venv/bin/activate        # Windows (PowerShell): .venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### שלב 3 — תלויות הדשבורד
```bash
cd ../evidence
npm install
```

### שלב 4 — הרצת ההדגמה (אנונימית, ללא אישורים)
```bash
cd ../pipeline
./run_all.sh --mock
```

לפי הסדר: נתוני הדגמה אנונימיים → KPI → NPV/IRR/MIRR/EAA/PI → **התאמת התפלגויות לטוקנים** →
**מונטה קרלו (10,000 איטרציות)** → AHP-TOPSIS 2n → **DEMATEL · ELECTRE · PROMETHEE · MAUT · MCDA-C** →
**חוסן הדירוג (דיריכלה)** → גרפים → תיק ניהולי → מפת 5D → pitch decks → בניית הדשבורד.

### שלב 5 — פתיחת הדשבורד
```bash
cd ../evidence
npm run dev          # http://localhost:3000
npm run preview      # (alternativa) serve o estático já compilado em build/
```

### שלב 6 — מעבר לנתונים שלכם

**6.1 — אישורים ופרמטרים** (הכול אופציונלי; בלי `.env` הפייפליין משתמש בברירות המחדל):
```bash
cd ../pipeline
cp .env.example .env      # edite: LANGFUSE_PUBLIC_KEY, LANGFUSE_SECRET_KEY, SELIC_ANUAL, USD_BRL...
```

**6.2 — תזרים המזומנים שלכם** (הוא שמזין את ה-NPV, ה-IRR ואת מונטה קרלו):
```bash
cp fluxo_caixa_template.csv fluxo_caixa.csv
```
מבנה: `periodo 0` היא ההשקעה (תזרים שלילי), ו-`taxa` הוא שיעור ההיוון לתקופה (`0.10` = 10%).
```csv
project_name,periodo,fluxo,taxa
Project A,0,-12000,0.10
Project A,1,3000,0.10
Project A,2,4000,0.10
```

**6.3 — הרצה עם נתונים אמיתיים:**
```bash
./run_all.sh          # sem --mock: sincroniza do Langfuse e usa fluxo_caixa.csv
```

עבור ה-pitch decks התקינו **tectonic** (למשל `cargo install tectonic` או מנהל החבילות של ההפצה שלכם).
